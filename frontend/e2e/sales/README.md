## 四、关键场景

### 1. 报价单创建
- 必填校验（客户/产品）
- 自动生成报价单号 `Q-YYYYMMDD-XXXX`
- 草稿可二次编辑
- 复制为新单

### 2. 销售订单创建
- 报价单 → 销售订单一键转换
- 双计量单位（米 + 公斤）
- 颜色/等级/潘通色号
- 动态行项增删

### 3. 销售订单审批
- 草稿 → 待审核
- 待审核 → 已审核
- 待审核 → 草稿/已驳回

### 4. 销售发货
- 创建发货单（自动生成 `DN-` 单号）
- 发货数量校验（≤ 订单数量）
- 部分发货 → 状态"部分发货"
- 全部发货 → 状态"已发货"

### 5. 应收单生成
- 已发货自动生成 AR 应收单
- 应收金额 = 发货金额 + 税额
- 分次收款

### 6. 销售收款
- 一次性收清
- 多次部分收款
- 5 种收款方式（银行转账/现金/承兑/支付宝/微信）
- 收款单打印

### 7. 销售统计
- 按月/客户/产品分组汇总
- 销售业绩 TOP 10
- 回款率
- 毛利率
- 退货率
- Excel 导出

## 五、技术约束

- **零硬编码**：所有数据通过 UI 交互输入，无 `process.env` 硬编码账号
- **URL 规范**：所有 URL 路径均遵循统一前缀（前端 vue-router 负责）
- **数据隔离**：每个 spec 用独立标识（如 `E2E 测试色卡`），避免与其他套件冲突
- **稳定性**：单测试 30s 超时，无重试，CI 上单线程运行
- **可读性**：每个 `test()` 都有中文 `01-XX` 编号 + 中文描述
