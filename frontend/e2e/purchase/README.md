## 四、关键场景

### 1. 采购订单创建
- 必填校验（供应商/产品）
- 自动生成采购单号 `PO-YYYYMMDD-XXXX`
- 交货日期 + 仓库
- 多产品行批量下单

### 2. 采购订单审批
- 草稿 → 待审核
- 待审核 → 已审核
- 待审核 → 草稿/已驳回

### 3. 采购入库
- 创建入库单（自动生成 `GR-` 单号）
- 入库数量校验（≤ 采购数量）
- 部分入库 → 状态"部分入库"
- 全部入库 → 状态"已入库"

### 4. 采购质检
- 入库单可发起质检（生成 `QI-` 单号）
- 质检合格 → 库存增加
- 质检不合格 → 触发退货

### 5. 应付单生成
- 已入库自动生成 AP 应付单
- 应付金额 = 入库金额 + 税额
- 分次付款

### 6. 采购付款
- 一次性付清
- 多次部分付款
- 5 种付款方式（银行转账/现金/承兑/支付宝/微信）
- 付款单打印

### 7. 供应商报表
- 采购汇总（按月/供应商）
- 到货及时率
- 质检合格率
- 应付账龄（30/60/90 天分段）
- 供应商评级（A/B/C/D）
- Excel 导出

## 五、技术约束

- **零硬编码**：所有数据通过 UI 交互输入
- **租户隔离**：URL 路径带 tenant 前缀
- **数据隔离**：E2E 测试标识独立（如 `E2E 测试：色差超标`）
- **稳定性**：单测试 30s 超时，零重试，CI 单线程
- **可读性**：中文 `01-XX` 编号 + 中文描述
