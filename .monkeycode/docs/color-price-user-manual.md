# 面料多色号定价扩展用户手册

> 冰溪 ERP P0-5 行业功能 - 面料多色号定价扩展
> 适用版本: P0-5 之后
> 更新日期: 2026-06-18

---

## 1. 简介

面料多色号定价是纺织行业 ERP 的核心功能。冰溪 ERP P0-5 在 P0-1 销售报价单 30% 基础上扩展到 100%，包括：

- **批量调价**：一次调整 50+ 色号（百分比 / 固定金额 / 阶梯价）
- **价格历史**：完整审计与回溯
- **阶梯定价**：数量越多价越低（4 档 + 客户等级叠加）
- **季节性调价**：春夏 / 秋冬 / 节日自动调价
- **客户专属价**：战略客户大客户协议价（最高优先级）
- **VIP 95 折**：客户等级全局叠加
- **价格预警**：>10% 自动审批 / >20% 涨跌幅预警
- **价格计算引擎**：4 档阶梯 + VIP 95 折 + 季节 + 客户专属 优先级统一计算

---

## 2. 角色与权限

| 角色 | 权限 |
|------|------|
| 销售员 | 查看价格、提交批量调价（≤10% 自动通过） |
| 销售经理 | 全部销售员权限 + 审批 10-30% 调价 |
| 总经理 | 全部权限 + 审批 >30% 调价 + 客户专属价管理 |
| 财务 | 查看所有价格 + 价格历史 |
| 管理员 | 全部权限 + 季节规则配置 |

## 3. 操作流程

### 3.1 维护基础价格

1. 进入 **色号价格** → **列表**
2. 点击 **新建价格**
3. 填写：
   - 产品 + 色号（必填）
   - 币种（CNY / USD / EUR）
   - 基础价（必填）
   - 生效日期（默认今天）
   - 客户等级（VIP / NORMAL / GOLD / SILVER，可选）
   - 季节（SS / AW / HOLIDAY，可选）
   - 优先级（数值大 = 优先级高）
4. 点击 **保存**

### 3.2 配置阶梯价

1. 进入 **色号价格** → **详情**
2. 在 **阶梯价** 区域点击 **添加阶梯**
3. 配置：
   - 起订量（min_quantity）
   - 数量上限（max_quantity，留空 = 无限）
   - 阶梯价
   - 客户等级（留空 = 通用）
   - 顺序（0=最低阶梯，10=最高阶梯）
4. 多次添加形成 4 档：
   - 1-99 米：100%（基础价）
   - 100-499 米：95%
   - 500-999 米：90%
   - 1000+ 米：85%
5. 点击 **保存**

### 3.3 设置季节调价规则

1. 进入 **色号价格** → **季节规则**
2. 点击 **新建规则**
3. 配置：
   - 规则名称
   - 季节（SS / AW / HOLIDAY）
   - 品类（留空 = 全部产品）
   - 调整方式（percentage / fixed）
   - 调整值（+0.10 = 涨 10%，-0.05 = 降 5%）
   - 生效日期
4. 点击 **保存**

### 3.4 客户专属价

1. 进入 **色号价格** → **客户专属价**
2. 点击 **新建专属价**
3. 选择客户 + 产品 + 色号
4. 设置专属价格或折扣率
5. 设置有效期
6. 提交 → 待总经理审批

### 3.5 批量调价

1. 进入 **色号价格** → **批量调价**
2. 多选色号
3. 选择调价模式：
   - **百分比**：+5% = 涨 5%
   - **固定金额**：+1.5 元/米
   - **阶梯价**：调整阶梯价
4. 填写调价原因
5. 点击 **提交**
6. 系统自动判断：
   - 涨跌幅 ≤ 10% → **自动通过**
   - 涨跌幅 > 10% → **待经理审批**

### 3.6 审批调价

1. 进入 **审批中心** → **待审批价格**
2. 查看调价详情（变更前/后价格、原因、操作人）
3. 选择 **通过** 或 **拒绝**
4. 添加审批意见
5. 提交

### 3.7 价格计算演示

1. 进入 **色号价格** → **批量调价**
2. 选中一个色号
3. 点击 **价格计算演示**
4. 系统返回：
   - 基础价
   - 阶梯价（如果命中）
   - 客户等级折扣（VIP 95 折）
   - 季节调价（如果在有效期内）
   - 客户专属价（最高优先级）
   - **最终价**
   - **应用规则链**（breakdown）

## 4. 价格计算公式

### 4.1 优先级（从高到低）

```
1. 客户专属价（customer_id 匹配 + 有效期）
2. 季节调价（在 active 规则区间内）
3. 阶梯价（min_quantity ≤ 数量 < max_quantity + 客户等级）
4. 客户等级折扣（VIP 95 折 / NORMAL 100%）
5. 基础价（fallback）
```

### 4.2 叠加公式

```
最终价 = 客户专属价（最高优先）
      = 基础价 × 阶梯折扣 × 客户等级折扣 × (1 + 季节百分比)
      = 基础价 + 季节固定金额
```

### 4.3 案例

- 基础价 = 100 元
- 数量 = 1000 米（命中阶梯 4 档 85%）
- 客户等级 = VIP（95 折）
- 季节 = 春夏（+10%）
- **最终价 = 100 × 0.85 × 0.95 × 1.10 = 88.83 元**

如果客户 A 还有专属价 80 元：
- **最终价 = 80 元**（最高优先级覆盖）

## 5. 价格预警规则

| 涨跌幅 | 行为 |
|--------|------|
| ≤ 10% | 自动通过，红色 PENDING 徽标 |
| 10% < x ≤ 20% | 需经理审批 |
| 20% < x ≤ 30% | 需经理审批 + 价格预警 |
| > 30% | 需总经理审批 + 价格预警 |
| < -30% | 价格预警（异常下跌） |

## 6. 常见问题

### Q1：批量调价后部分色号失败？

答：调价失败的色号会保留在原状态，详情页可查看历史。

### Q2：阶梯价与季节调价冲突？

答：系统按优先级应用：客户专属 > 季节 > 阶梯 > 等级 > 基础价。

### Q3：客户专属价什么时候生效？

答：调价审批通过后立即生效，详情页可见 `approval_status = APPROVED`。

### Q4：删除色号价格后历史还在吗？

答：删除是软删除（`is_active = false`），历史记录完整保留。

### Q5：如何调整价格的优先级？

答：详情页修改 `priority` 字段，数值大 = 优先级高。
