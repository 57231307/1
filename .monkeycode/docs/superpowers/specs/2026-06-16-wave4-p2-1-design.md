## 一、目标与背景

### 1.1 业务目标

将 Element Plus 2.6+ 内置的虚拟滚动表格组件 `el-table-v2` 落地到 4 个核心业务页面，替换传统 `el-table`，解决大数据量表格卡顿问题。

### 1.2 技术目标

- 4 个页面（库存台账 / 销售订单 / 生产工单 / 质检记录）在真实数据下首屏渲染 < 300ms
- 抽出 `V2Table` 通用组件 + `useTableApi` composable，覆盖 4 页面表格场景
- 100+ `el-table` 使用页面中，4 个高价值页面优先升级
- 为后续 P2-2 性能优化（数据库查询 + 缓存）铺路

### 1.3 范围

**包含**（5 PR 串行）：
1. PR-1：抽 `useTableApi` composable + `V2Table` 组件 + 4-5 单元测试
2. PR-2：迁移 `inventory/tabs/StockTab.vue`（244 行）
3. PR-3：迁移 `sales/views/OrderListView.vue`
4. PR-4：迁移 `production/index.vue`（532 行）
5. PR-5：迁移 `quality/tabs/RecordTab.vue` + **删除 5 个废弃文件**

**不包含**：
- 性能基准测试（无 mock 数据，CI 不可量化）
- mock 数据生成
- Lighthouse CI
- 移动端适配优化（留待 P2-3）
- 拖拽排序 / 多选 / 序号列等高级特性（如 4 页面需要则走 issue 评估）

## 四、组件 API

### 4.1 V2Table 组件 props

```typescript
// components/V2Table/types.ts
import type { VNode } from 'vue'

export type SortOrder = 'asc' | 'desc'

export interface ColumnDef {
  key: string                              // 数据字段
  title: string                            // 列标题
  width?: number                           // 固定宽度
  minWidth?: number                        // 最小宽度
  fixed?: 'left' | 'right'                 // 固定列
  sortable?: boolean                       // 是否可排序
  align?: 'left' | 'center' | 'right'      // 对齐
  formatter?: (row: any) => string         // 格式化函数
  renderCell?: (row: any) => VNode         // 自定义渲染
  hidden?: boolean                         // 是否隐藏
}

export interface V2TableProps {
  columns: ColumnDef[]
  data: any[]
  loading?: boolean
  total?: number                           // 总数
  page?: number
  pageSize?: number
  pageSizes?: number[]                     // 默认 [10, 20, 50, 100]
  height?: number | string                 // 表格高度，默认 600px
  rowKey?: string                          // 默认 'id'
  emptyText?: string
  onPageChange?: (page: number) => void
  onSizeChange?: (size: number) => void
  onSortChange?: (key: string, order: SortOrder) => void
  onRowClick?: (row: any) => void
}
```

### 4.2 useTableApi composable

```typescript
// composables/useTableApi.ts
import type { Ref } from 'vue'

export interface UseTableApiOptions<T = any> {
  url: string
  defaultParams?: Record<string, any>
  defaultPageSize?: number                  // 默认 20
  pageKey?: string                          // 默认 'page'
  pageSizeKey?: string                      // 默认 'page_size'
  totalKey?: string                         // 默认 'total'
  listKey?: string                          // 默认 'list' / 'items' / 'data'（依次探测）
  retryCount?: number                       // 默认 2
  retryDelay?: number                       // 默认 1000ms
  onError?: (err: any) => void
}

export interface UseTableApiReturn<T = any> {
  data: Ref<T[]>
  total: Ref<number>
  loading: Ref<boolean>
  page: Ref<number>
  pageSize: Ref<number>
  queryParams: Ref<Record<string, any>>
  refresh: () => Promise<void>
  reset: () => void
  setQueryParam: (key: string, value: any) => void
}

export function useTableApi<T = any>(
  options: UseTableApiOptions<T> | string
): UseTableApiReturn<T>
```

**字符串重载**：`useTableApi<T>('/inventory/stocks')` 等价于 `useTableApi<T>({ url: '/inventory/stocks' })`

### 4.3 4 页面典型调用方式

```vue
<!-- inventory/tabs/StockTab.vue（PR-2 迁移示例）-->
<script setup lang="ts">
import { useTableApi } from '@/composables/useTableApi'
import V2Table from '@/components/V2Table/index.vue'

interface Stock {
  id: number
  product_code: string
  product_name: string
  warehouse_name: string
  quantity: number
  status: 'normal' | 'warning' | 'frozen'
}

const { data, loading, page, pageSize, total, refresh, queryParams } = useTableApi<Stock>('/inventory/stocks')

const columns = [
  { key: 'product_code', title: '产品编码', width: 140, fixed: 'left' as const },
  { key: 'product_name', title: '产品名称', minWidth: 180 },
  { key: 'warehouse_name', title: '仓库', width: 120 },
  { key: 'quantity', title: '数量', width: 100, sortable: true, align: 'right' as const },
  {
    key: 'status',
    title: '状态',
    width: 100,
    formatter: (row: Stock) => {
      const map = { normal: '正常', warning: '预警', frozen: '冻结' }
      return map[row.status] || row.status
    },
  },
]
</script>

<template>
  <div class="stock-tab">
    <!-- 顶部筛选 / 操作按钮保持原样 -->
    <el-card class="filter-card">...</el-card>

    <!-- 表格替换为 V2Table -->
    <el-card class="table-card">
      <V2Table
        :columns="columns"
        :data="data"
        :loading="loading"
        :page="page"
        :page-size="pageSize"
        :total="total"
        :height="600"
        @page-change="page = $event"
        @size-change="pageSize = $event"
        @refresh="refresh"
      />
    </el-card>
  </div>
</template>
```

## 七、风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| el-table-v2 不支持 4 页面某些特性 | 迁移失败 | PR 实施前预检，不支持特性 throw + known issues 记录，不留半成品 |
| 真实数据量下性能不达预期 | 用户体验差 | 浏览器手动验收（按 Q9），不达预期则回退该 PR（git revert）|
| 4 页面列定义差异大，组件 API 频繁扩展 | 组件维护成本高 | PR-1 设计 API 时考虑 4 页面共性 + 预留 renderCell 扩展点 |
| 删除 POC 路径导致外链失效 | 文档失链 | 检查是否有外链引用（如有则更新文档）|
| 删除 DraggableTable 失去 demo 教学 | 教学资源减少 | components-demo 路由保留，其他 3 Tab 正常使用 |
| PR 数量多（5 个），CI 排队时间长 | 交付延迟 | 串行调度避免冲突，每个 PR 必须 4 job 全绿才合并 |

### 7.1 已知不支持特性（known issues）

| 特性 | el-table-v2 状态 | 处理 |
|------|------------------|------|
| 拖拽排序（Sortable.js）| 不内置支持 | 如 4 页面需要，throw + 记录 issue |
| 多选（type="selection"）| 不内置支持，需 renderCell 自定义 | 4 页面当前未使用，throw + 记录 issue |
| 序号列（type="index"）| 不内置支持，需 renderCell 自定义 | 4 页面当前未使用，throw + 记录 issue |
| 树形数据 | 不支持 | 4 页面未使用，不在范围 |
| 列合并 / 行合并 | 不支持 | 4 页面未使用，不在范围 |
