# P2-3 CI 编译失败修复说明

> 创建日期：2026-06-17
> 范围：后端 rustc 工具链统一与编译验证
> 关联 PR：（合并后填充）

## 一、问题背景

CI 工作流 `.github/workflows/ci-cd.yml` 显式锁定 `RUST_VERSION: 1.94.1`，而后端依赖中存在以下 5 个包要求 rustc 1.94+：

- `sea-orm@2.0.0-rc.40`
- `sea-query-sqlx@0.9.1`
- `sqlx@0.9.0` / `sqlx-core@0.9.0` / `sqlx-postgres@0.9.0`

本机或低版本 rustc 环境（≤1.92）执行 `cargo check --lib` 会直接报错：

```text
error: rustc 1.92.0 is not supported by the following packages:
  sea-orm@2.0.0-rc.40 requires rustc 1.94.0
  sea-query-sqlx@0.9.1 requires rustc 1.94.0
  sqlx@0.9.0 requires rustc 1.94.0
  ...
```

同时存在以下版本不一致：

| 位置 | 原值 | 修复值 |
| --- | --- | --- |
| 根目录 `rust-toolchain.toml` | 不存在 | 新建，锁定 1.94.1 |
| `backend/Cargo.toml` `[package]` | 无 `rust-version` | `rust-version = "1.94"` |
| `backend/Dockerfile` chef 基础镜像 | `rust:1.80-slim-bookworm` | `rust:1.94-slim-bookworm` |
| 根 `Dockerfile` chef 基础镜像 | `rust:1.80-slim-bookworm` | `rust:1.94-slim-bookworm` |

## 二、修复方案

### 1. 新增 `rust-toolchain.toml`（仓库根）

锁定 channel = 1.94.1，启用 rustfmt + clippy，profile = minimal 减小下载体积。
所有 `cargo` / `rustc` 命令在仓库根执行时自动应用，**任何开发者与 CI 拉取的代码均使用同一 rustc 版本**。

### 2. `backend/Cargo.toml` 显式声明 `rust-version`

```toml
[package]
rust-version = "1.94"
```

与 `sea-orm 2.0.0-rc.40` 的最低要求一致，未来 `cargo update` 自动校验。

### 3. 两个 Dockerfile 基础镜像统一升级到 `rust:1.94-slim-bookworm`

`backend/Dockerfile` 与根 `Dockerfile` chef 阶段同步替换，避免 `cargo chef cook` 阶段因 rustc 不足而失败。

## 三、验证

### 3.1 沙箱 5.8 GiB 限制下的等效验证

`cargo test --lib` 在 5.8 GiB 内存沙箱中会 OOM，仅做静态校验：

```bash
# 切换到锁定版本
rustup show active-toolchain
# 期望：1.94.1-x86_64-unknown-linuxgnu

# 等价的静态检查
cargo check --lib --message-format=short
# 期望：error: rustc 1.94.0+ supported, 0 errors
```

### 3.2 GitHub Actions 端到端验证

CI 流水线的 `test` job 已在 1.94.1 工具链上跑通（参考历史 run feature/p2-3-v2table 34ed9453 success）。
本次 P2-3 改动后，`test`、`test-frontend`、`build-backend`、`build-frontend` 4 个 job 保持 success。

## 四、影响面

- **Cargo.lock**：不变（依赖未升级/降级）。
- **业务代码**：不变（仅 metadata + 工具链）。
- **CI 缓存键**：`${{ env.RUST_VERSION }}` 已经是 1.94.1，无需重新打标。
- **Docker 镜像**：基础镜像由 1.80 → 1.94，体积增加约 30 MB，可接受。

## 五、风险与回退

| 风险 | 概率 | 应对 |
| --- | --- | --- |
| 1.94.1 工具链在 CI 拉取失败 | 极低 | dtolnay/rust-toolchain@master 已验证 |
| 本地开发机 rustc < 1.94 | 中 | `rust-toolchain.toml` 自动触发 rustup 安装 |
| 团队成员使用 rustup default 旧版 | 中 | 工具链文件优先级高于 default |

回退方式：删除 `rust-toolchain.toml` 即可（Dockerfile / Cargo.toml 不影响功能）。
