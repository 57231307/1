## 二、问题分类

### 2.1 按严重程度分类

**P0（必须立即修复）**：
1. 财务凭证create方法无事务
2. 财务凭证辅助字段丢失

**P1（高优先级）**：
1. 采购订单外键验证缺失
2. 销售订单created_by未保存
3. 库存并发控制缺失
4. 财务分页错误
5. 财务并发余额更新

**P2（中优先级）**：
1. 库存流水记录不完整
2. 生产订单外键验证缺失
3. CRM owner_name硬编码
4. 应收发票无自动联动

### 2.2 按问题类型分类

**数据验证问题**：
- 外键存在性未验证（采购、销售、生产、CRM）
- 必填字段默认值不合理（生产、采购）

**事务处理问题**：
- 关键操作无事务（财务create）
- 事务回滚错误被忽略（采购、销售）
- 部分操作未纳入事务（CRM更新）

**并发控制问题**：
- 库存超卖风险（库存、销售）
- 订单号重复风险（生产、采购）
- 状态更新竞态（生产、CRM）

**数据完整性问题**：
- 辅助字段丢失（财务）
- 库存流水记录缺失（库存）
- 部分字段未保存（采购line_no）

## 三、修复优先级

### 第一阶段：P0问题修复（1天）

1. **财务凭证create方法添加事务**
   - 将主表和分录插入包裹在事务中
   - 失败时自动回滚

2. **财务凭证辅助字段保存**
   - 从请求中读取辅助核算字段
   - 传递给Service层

### 第二阶段：P1问题修复（2天）

3. **采购订单外键验证**
   - 验证warehouse_id、department_id、product_id存在性

4. **销售订单created_by保存**
   - Handler层传递user_id
   - Service层设置created_by

5. **库存并发控制**
   - 添加乐观锁（version字段）
   - 或使用悲观锁（SELECT FOR UPDATE）

6. **财务分页修复**
   - 修正offset计算公式

7. **财务并发余额更新**
   - 使用SELECT FOR UPDATE保护余额更新

### 第三阶段：P2问题修复（2天）

8. **库存流水记录补全**
   - 调拨发出/接收记录流水
   - 盘点调整记录流水
   - 库存调整记录流水

9. **生产订单验证增强**
   - 添加外键验证
   - 添加数量验证

10. **CRM优化**
    - 修复owner_name硬编码
    - 添加customer_id验证

11. **应收发票自动联动**
    - 参照应付实现销售出库自动生成

    db: &DatabaseConnection,
    entity: T,
    id: i32,
    name: &str,
) -> Result<(), AppError> {
    let exists = entity.find_by_id(id).one(db).await?;
    if exists.is_none() {
        return Err(AppError::BadRequest(format!("{} ID {} 不存在", name, id)));
    }

}

### 5.2 风险评估

- **数据一致性风险**：高（财务无事务、库存无锁）
- **业务逻辑风险**：中（外键验证缺失、字段未保存）
- **并发安全风险**：高（库存超卖、订单号重复）

### 5.3 建议

1. **立即修复**：财务凭证事务和辅助字段问题
2. **尽快修复**：库存并发控制和外键验证
3. **计划修复**：库存流水记录和应收自动联动
