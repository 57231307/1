## 排查概述

本报告聚焦功能问题、功能可用性、功能孤岛、功能完善、功能完整、实现完整性等方面的问题。

## 一、功能可用性问题

### 1.1 关键功能不可用

| 问题 | 严重程度 | 影响范围 | 说明 |
|------|----------|----------|------|
| system/index.vue导入错误 | **严重** | 系统管理页面 | 默认导入vs命名导出不匹配，导致页面崩溃 |
| generate-no端点缺失 | **严重** | 库存/采购页面 | 4个页面调用不存在的API端点 |
| 未注册路由的页面 | 中等 | 2个页面 | sales-analysis和security页面无法访问 |

### 1.2 问题详情

#### system/index.vue导入错误
```typescript
// 错误代码（第668行）
import request from '@/api/request'  // 默认导入

// 正确代码
import { request } from '@/api/request'  // 命名导入
```

**影响**：系统管理页面完全不可用，包括：
- 权限列表获取
- 数据权限获取
- 字段权限获取
- 通知设置获取
- 审计日志获取
- Webhook集成CRUD
- 系统更新检查
- 租户配置CRUD

#### generate-no端点缺失
| 前端页面 | 调用的端点 | 后端是否存在 |
|----------|------------|--------------|
| inventoryCount/index.vue | /inventory-count/generate-no | ❌ |
| purchaseReceipt/index.vue | /purchase-receipt/generate-no | ❌ |
| inventoryAdjustment/index.vue | /inventory-adjustment/generate-no | ❌ |
| inventoryTransfer/index.vue | /inventory-transfer/generate-no | ❌ |

**影响**：这些页面在创建新记录时无法自动生成单据编号。

## 二、功能孤岛问题

### 2.1 事件驱动断裂

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| MaterialShortageAlert事件未定义 | **严重** | 事件在material_shortage_service中发布，但event_bus中未定义 |
| 事件处理逻辑缺失 | 高 | 多个事件仅记录日志，无后续处理 |

### 2.2 模块集成断裂

| 集成点 | 现状 | 期望 | 严重程度 |
|--------|------|------|----------|
| 采购入库→库存 | 仅更新订单状态 | 更新状态+增加库存+记录流水 | **严重** |
| 销售发货→财务 | 仅扣减库存 | 扣减库存+生成应收账款 | **严重** |
| 生产完成→库存 | 仅更新状态 | 更新状态+入库成品+扣减原材料 | **严重** |
| 库存变动→财务 | 无联动 | 自动生成会计凭证 | 高 |
| CRM商机→销售 | 无明细创建 | 创建订单明细 | 中 |

### 2.3 数据孤岛

| 数据孤岛 | 影响模块 | 说明 |
|----------|----------|------|
| 财务指标缺少库存价值 | 财务+库存 | 无法计算存货周转率 |
| 库存变动未生成凭证 | 库存+财务 | 财务数据不完整 |
| 盘点差异未触发调整 | 库存 | 需要手动处理 |
