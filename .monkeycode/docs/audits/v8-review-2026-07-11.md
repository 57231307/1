## 三、详细问题清单

### 🔴 高风险（4 项，必须修复）

#### H1. unwrap_or_default() 静默丢弃 SSRF 防护
- **文件**：[webhook_service.rs:217](file:///workspace/backend/src/services/webhook_service.rs#L217)
- **问题**：`reqwest::Client::builder()...build().unwrap_or_default()`，build 失败时创建默认客户端，丢弃 `redirect(Policy::none())` 和 `resolve_to_addrs` 配置，SSRF 防护被静默绕过
- **修复**：改为 `.map_err(|e| AppError::internal(...))?`（参考 currency_service.rs:310）

#### H2. validate_dir_recursive 缺少递归深度限制
- **文件**：[backup.rs:15-34](file:///workspace/backend/src/cli/util/backup.rs#L15-L34)
- **问题**：递归校验无深度上限，恶意 tar 包含数千层嵌套目录可导致栈溢出
- **修复**：添加 `depth` 参数，上限 100 层

#### H3. 临时目录硬编码且可预测
- **文件**：[backup.rs:134](file:///workspace/backend/src/cli/util/backup.rs#L134)
- **问题**：`/tmp/bingxi_restore` 固定路径，存在符号链接竞争攻击（TOCTOU）
- **修复**：使用 `mktemp` 或随机生成的临时目录名

#### H4. 日志泄露完整数据库/缓存 URL（含凭据）
- **文件 1**：[app_state.rs:332](file:///workspace/backend/src/utils/app_state.rs#L332) - 记录 `ELASTICSEARCH_URL`
- **文件 2**：[rate_limit.rs:158](file:///workspace/backend/src/middleware/rate_limit.rs#L158) - 记录 `RATE_LIMIT_REDIS_URL`
- **问题**：URL 可能含 `user:password@host` 格式的凭据
- **修复**：只记录"已配置"，不记录实际 URL（参考 auth_service.rs:436）

### 🟡 中风险（8 项，应修复）

#### M1. download_update 缺少 resolve_to_addrs（DNS Rebinding）
- **文件**：[system_update_service.rs:735-739](file:///workspace/backend/src/services/system_update_service.rs#L735-L739)
- **问题**：有域名白名单和重定向限制，但无 `resolve_to_addrs` 固定 IP
- **修复**：复用 `validate_url_and_resolve` + `resolve_to_addrs`

#### M2. Elasticsearch 客户端无 SSRF 防护
- **文件**：[elastic.rs:279-282](file:///workspace/backend/src/search/elastic.rs#L279-L282) 和 [elastic.rs:631-634](file:///workspace/backend/src/search/elastic.rs#L631-L634)
- **问题**：无 `redirect(Policy::none())`，无 `resolve_to_addrs`
- **修复**：添加重定向限制

#### M3. Python 代码拼接密码
- **文件**：[admin.rs:33-52](file:///workspace/backend/src/cli/admin.rs#L33-L52)
- **问题**：密码通过字符串拼接嵌入 Python 代码，仅转义双引号
- **修复**：通过 stdin 或环境变量传递密码

#### M4. 后置校验存在 TOCTOU 风险
- **文件**：[backup.rs:146-157](file:///workspace/backend/src/cli/util/backup.rs#L146-L157)
- **问题**：tar 解压后才校验路径，恶意文件在校验前已写入磁盘
- **修复**：改为逐文件校验（参考 system_update_service.rs:412-422）

#### M5. 硬编码系统路径
- **文件**：[backup.rs:81-82](file:///workspace/backend/src/cli/util/backup.rs#L81-L82)
- **问题**：`/etc/bingxi/.env` 和 `/etc/systemd/system/` 硬编码
- **修复**：通过配置或环境变量管理

#### M6. 限流器不支持分布式部署
- **文件**：[webhook_handler.rs:18-19](file:///workspace/backend/src/handlers/webhook_handler.rs#L18-L19)
- **问题**：`MemoryRateLimiter` 纯内存，多实例实际阈值为 `10 * N`
- **修复**：使用 `check_rate_limit` 函数（Redis 优先 + 内存回退）

#### M7. 硬编码 API URL
- **文件**：[currency_service.rs:293-296](file:///workspace/backend/src/services/currency_service.rs#L293-L296)
- **问题**：`https://api.exchangerate-api.com` 硬编码
- **修复**：通过环境变量或配置管理

#### M8. 6 个修改文件全部无单元测试
- **文件**：tracking_service.rs / backup.rs / currency_service.rs / webhook_service.rs / webhook_handler.rs / system_update_service.rs
- **问题**：违反规则 5"修改现有功能需要更新相关测试"
- **修复**：为每个修复点添加单元测试
