### 2.2 P0 核心问题归纳（8 大类）

#### 2.2.1 面料行业核心特性缺失（dye_batch.dye_lot_no + 色卡发放 + 大货批色）
- **dye_batch 表缺少 dye_lot_no 字段**（类四 P0）：影响四层级联/成本归集/缸号追溯
- **色卡发放专项完全未实现**（类十 14 P0）：后端/前端/数据库/路由/校验/权限/定时任务/数据迁移全部缺失
- **大货批色业务模块完全缺失**（类十一 5 P0）：bulk_color_approval 表不存在，ship_order 不校验批色状态

#### 2.2.2 安全与权限严重缺陷（行级数据权限 + IDOR + 超级权限注入）
- **行级数据权限完全未实现**（类十二+类十四+类十八 P0）：销售能查询所有客户订单，CRM 无数据权限
- **越权访问防护 IDOR 未实现**（类十二 P0）：所有 /:id 路由未校验资源归属
- **`*:*` 超级权限注入**（类十四 P0）：is_system 角色获得超级权限
- **14 类业务角色缺失**（类十四 P0）：仅 admin/manager/operator 3 个角色

#### 2.2.3 打印导出审计与安全缺陷
- **业务级审计覆盖率仅 23%**（类十三 P0）：10 个导出端点完全无审计
- **2 个敏感端点缺 AuthContext**（类十三 P0）：染色配方/缸号导出无认证
- **25+ 前端本地导出完全无审计**（类十三 P0）：纯前端 HTML 生成绕过后端
- **审计日志导出"假按钮"陷阱**（类十三 P0）：前端走本地 exportToExcel 而非后端 API
- **打印 HTML 是占位假数据**（类十三 P0）：硬编码 stub 数据

#### 2.2.4 测试体系严重缺陷
- **核心 service 零单元测试**（类六 P0）：quotation_service / purchase_receipt_service 零测试
- **7 项关键业务路径无集成测试**（类六 P0）：生产订单/采购收货/销售发货/付款/染整/打样/大货处方
- **CI baseline 掩盖编译失败**（类六 P0）：bi_analysis_test.rs 16 个测试 API 脱节但 CI 通过
- **E2E 通过率 0%**（类六 P0）：远低于 90% 要求
- **CI 不集成覆盖率工具**（类六 P0）：无 cargo tarpaulin / codecov

#### 2.2.5 部署与运维违规
- **Docker 违规**（类七 P0）：4 个 Docker 文件违反禁止 Docker 规则
- **PostgreSQL 客户端安装违规**（类七 P0）：install.sh 安装 postgresql-client
