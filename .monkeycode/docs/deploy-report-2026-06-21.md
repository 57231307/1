## 二、发现的问题

### 问题 1：配置文件备份不存在

**严重程度**：中
**错误信息**：
```
cp: cannot stat 'config.yaml.bak': No such file or directory
cp: cannot stat '.env.bak': No such file or directory
```

**影响**：
- 无法恢复之前的配置文件
- 可能导致服务使用默认配置启动

**原因**：
- 之前的部署没有创建备份文件
- 配置文件可能已被删除

**建议**：
- 部署前应先备份当前配置文件
- 创建标准化的部署脚本，包含备份步骤

---

### 问题 2：数据库迁移失败（tenant_id 列不存在）

**严重程度**：高
**错误信息**：
```
WARN server: 启动时核心迁移失败: Execution Error: error returned from database: column "tenant_id" does not exist，将在初始化时重试
```

**影响**：
- 核心迁移未完成
- 可能影响多租户功能

**原因**：
- 数据库 schema 与代码不匹配
- 可能缺少迁移文件或迁移未执行

**建议**：
- 检查数据库中 omni_audit_logs 表结构
- 手动执行缺失的迁移
- 验证所有迁移文件是否完整

---

### 问题 3：pg_stat_statements 扩展不可用

**严重程度**：低
**错误信息**：
```
WARN server::services::slow_query_collector: 慢查询首屏采集失败（可能是 pg_stat_statements 不可用）: Query Error: error returned from database: relation "pg_stat_statements" does not exist
```

**影响**：
- 慢查询采集功能不可用
- 不影响核心业务功能

**原因**：
- PostgreSQL 未安装 pg_stat_statements 扩展
- 或扩展未启用

**建议**：
- 在数据库服务器上启用 pg_stat_statements 扩展
- 或在配置中禁用慢查询采集功能

---

### 问题 4：健康检查端点返回 404

**严重程度**：中
**错误信息**：
```
WARN server: 请求异常 status=404 Not Found latency_ms=0
```

**影响**：
- 无法通过 /health 端点检查服务健康状态
- 可能影响负载均衡器和监控系统

**原因**：
- 健康检查端点可能不存在
- 或端点路径不正确

**建议**：
- 检查后端代码中健康检查端点的实现
- 确认端点路径是否正确
- 添加或修复健康检查端点
