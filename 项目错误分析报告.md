## 一、编译错误统计与分类

### 1.1 后端编译错误（总计约 300+ 错误）

#### 错误类型 A：类型定义错误（RustDecimal vs Decimal）- 8 个文件
**错误代码**: E0412 - cannot find type `RustDecimal` in this scope

**影响文件**:
1. `backend\src\models\purchase_receipt.rs` - 3 处 (行 56, 60, 64)
2. `backend\src\models\ap_invoice.rs` - 6 处 (行 55, 59, 63, 75, 78, 82)
3. `backend\src\models\ap_payment.rs` - 使用别名导入
4. `backend\src\models\ap_payment_request.rs` - 使用别名导入
5. `backend\src\models\ap_payment_request_item.rs` - 使用别名导入
6. `backend\src\models\ap_verification.rs` - 2 处 (行 22, 32)
7. `backend\src\models\ap_verification_item.rs` - 2 处 (行 27, 29)
8. `backend\src\models\ap_reconciliation.rs` - 6 处

**根本原因**: 
- 部分模型文件直接使用 `RustDecimal` 类型但未导入
- 部分文件使用 `use rust_decimal::Decimal as RustDecimal;` 导入别名
- 应该统一使用 `Decimal` 类型并正确导入

**解决方案**:
```rust
// 在模型文件顶部添加导入
use rust_decimal::Decimal;

// 将所有 RustDecimal 替换为 Decimal
pub amount: Decimal,  // 而不是 RustDecimal
```

#### 错误类型 B：缺少 Set 函数导入 - 主要集中在 Service 层
**错误代码**: E0425 - cannot find function, tuple struct or tuple variant `Set` in this scope

**影响文件**:
- `backend\src\services\purchase_receipt_service.rs` - 100+ 处错误

**根本原因**:
- SeaORM 的 ActiveModel 需要使用 `ActiveValue::Set` 来设置字段值
- 缺少 `use sea_orm::ActiveValue::Set;` 导入

**解决方案**:
```rust
// 在服务文件顶部添加导入
use sea_orm::{
    ActiveModelTrait, ColumnTrait, DatabaseConnection, EntityTrait, QueryFilter,
    QueryOrder, TransactionTrait, ActiveValue::Set,  // 添加这一行
};
```

#### 错误类型 C：日期类型缺失
**错误代码**: E0412 - cannot find type `NaiveDate` in this scope

**影响文件**:
- `backend\src\models\ap_verification.rs` - 行 22
- `backend\src\models\ap_reconciliation.rs` - 行 27

**解决方案**:
```rust
use chrono::NaiveDate;  // 确保导入
```

### 1.2 前端编译错误（总计 169 个错误）

#### 错误类型 A：Yew Router API 变更 - 影响所有页面组件
**错误代码**: E0277, E0599 - trait bound not satisfied, method not found

**主要问题**:
1. `yew_router::BrowserRouter` 不再实现 `yew::Component` trait
2. `navigator` 方法在 Yew 0.21 版本中已被移除
3. `dyn_into` 方法需要导入 `wasm_bindgen::cast::JsCast`

**影响文件** (所有页面组件):
- `frontend\src\app\mod.rs` - 路由配置错误
- `frontend\src\pages\login.rs` - 导航方法错误
- `frontend\src\pages\user_list.rs` - 导航方法错误
- `frontend\src\pages\role_list.rs` - 导航方法错误
- `frontend\src\pages\product_list.rs` - 导航方法错误
- `frontend\src\pages\warehouse_list.rs` - 导航方法错误
- `frontend\src\pages\department_list.rs` - 导航方法错误
- `frontend\src\pages\product_category.rs` - 导航方法错误
- `frontend\src\pages\inventory_stock.rs` - 导航方法错误
- `frontend\src\pages\inventory_transfer.rs` - 导航方法错误
- `frontend\src\pages\inventory_count.rs` - 导航方法错误
- `frontend\src\pages\finance_invoice.rs` - 导航方法错误
- `frontend\src\pages\sales_order.rs` - 导航方法错误

**根本原因**:
- Yew 从 0.20 升级到 0.21 后，Router API 发生重大变更
- 旧的 `use_navigator()` hook 已被废弃
- 需要使用新的 `yew_router::use_route` 和 `yew_router::Link` 组件

**解决方案**:
```rust
// 旧代码 (Yew 0.20)
let navigator = use_navigator().unwrap();
navigator.push(Route::ProductList);

// 新代码 (Yew 0.21)
let navigator = use_navigator();
if let Some(navigator) = navigator {
    navigator.push(Route::ProductList);
}

// 或者使用 Link 组件
html! {
    <Link<Route> to={Route::ProductList}>{"产品列表"}</Link<Route>>
}
```
