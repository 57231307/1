# 秉羲 Rust ERP 系统 - 编译错误快速修复指南

## 修复优先级：🔴 P0（必须立即修复）

### 1. 后端 RustDecimal 类型错误（8 个文件）

**影响文件**:
- `backend/src/models/purchase_receipt.rs`
- `backend/src/models/ap_invoice.rs`
- `backend/src/models/ap_payment.rs`
- `backend/src/models/ap_payment_request.rs`
- `backend/src/models/ap_payment_request_item.rs`
- `backend/src/models/ap_verification.rs`
- `backend/src/models/ap_verification_item.rs`
- `backend/src/models/ap_reconciliation.rs`

**修复步骤**:

在每个文件的顶部添加导入（在 `use sea_orm::entity::prelude::*;` 之后）:
```rust
use rust_decimal::Decimal;
```

然后将所有 `RustDecimal` 替换为 `Decimal`:
```rust
// 修改前
pub amount: RustDecimal,

// 修改后
pub amount: Decimal,
```

**快速修复命令**（使用 IDE 的查找替换功能）:
- 查找：`RustDecimal`
- 替换：`Decimal`
- 范围：上述 8 个文件

### 2. 后端 Set 导入缺失（Service 层）

**影响文件**:
- `backend/src/services/purchase_receipt_service.rs` (100+ 错误)
- 可能还有其他 Service 文件

**修复步骤**:

在 Service 文件顶部修改导入：
```rust
// 修改前
use sea_orm::{
    ActiveModelTrait, ColumnTrait, DatabaseConnection, EntityTrait, QueryFilter,
    QueryOrder, TransactionTrait,
};

// 修改后
use sea_orm::{
    ActiveModelTrait, ColumnTrait, DatabaseConnection, EntityTrait, QueryFilter,
    QueryOrder, TransactionTrait, ActiveValue::Set,
};
```

**快速修复命令**:
在所有包含 `Set(` 的 Service 文件中，在导入语句中添加 `ActiveValue::Set,`

- `frontend/src/pages/financial_analysis.rs`
- `frontend/src/pages/supplier_evaluation.rs`

**修复步骤**:

#### 4.1 导入 JsCast trait（用于 dyn_into）

在每个页面文件顶部添加：
```rust
use wasm_bindgen::cast::JsCast;
```

#### 4.2 修复 navigator 调用

**修改前**（Yew 0.20 风格）:
```rust
let navigator = use_navigator().unwrap();
navigator.push(Route::ProductList);
```

**修改后**（Yew 0.21 风格）:
```rust
let navigator = use_navigator();
if let Some(nav) = navigator {
    nav.push(Route::ProductList);
}
```

或者使用 Link 组件替代编程式导航：
```rust
html! {
    <Link<Route> to={Route::ProductList}>
        <button>{"产品列表"}</button>
    </Link<Route>>
}
```

#### 4.3 修复 app/mod.rs 的路由配置

**修改前**:
```rust
use yew_router::prelude::*;

html! {
    <BrowserRouter>
        <Switch<Route> render={switch} />
    </BrowserRouter>
}
```

**修改后**（Yew 0.21）:
```rust
use yew_router::prelude::*;

html! {
    <HashRouter>
        <Switch<Route> render={switch} />
    </HashRouter>
}
```

或使用 BrowserRouter（如果支持）:
```rust
html! {
    <BrowserRouter>
        <Main />
    </BrowserRouter>
}
```

## 常见问题

### Q1: 修复后仍然有编译错误怎么办？
A: 运行 `cargo check --message-format=short` 查看详细错误，逐个修复。

### Q2: 前端编译通过但运行时报错？
A: 检查浏览器控制台，可能是运行时错误而非编译错误。

### Q3: 后端启动失败？
A: 检查数据库连接配置，确保 PostgreSQL 运行正常。
