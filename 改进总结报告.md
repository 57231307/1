```rust
pub async fn verify_invoice(&self, id: i32, paid_date: DateTime<Utc>, payment_method: String) -> Result<InvoiceModel, DbErr> {
    // 开启事务
    let txn = self.db.begin().await?;
    
    let mut invoice: ActiveModel = FinanceInvoice::find_by_id(id)
        .one(&txn)
        .await?
        .ok_or(DbErr::RecordNotFound(format!("发票 {} 不存在", id)))?
        .into();

    invoice.status = Set("verified".to_string());
    invoice.paid_date = Set(Some(paid_date));
    invoice.payment_method = Set(Some(payment_method));
    invoice.updated_at = Set(Utc::now());

    let result = invoice.update(&txn).await?;
    
    // 提交事务
    txn.commit().await?;
    
    Ok(result)
}
```

**改进效果**:
- ✅ 发票核销操作现在具有原子性
- ✅ 失败时自动回滚，避免数据不一致

### 3. ✅ 前端添加简单重试机制

**改进文件**:
- `e:\1\10\bingxi-rust\frontend\src\services\api.rs` (新建)
- `e:\1\10\bingxi-rust\frontend\Cargo.toml`

**改进内容**:
- 创建统一的 `ApiService` 模块
- 实现带重试机制的 HTTP 请求方法
- 支持 GET、POST、PUT、DELETE 方法
- 指数退避策略：1s, 2s, 4s
- 最大重试次数：3 次

**新增依赖** (`Cargo.toml`):
```toml
gloo-timers = "0.3"
```

**核心实现** (`api.rs`):
```rust
/// API 服务
/// 提供统一的 HTTP 请求方法，支持自动重试
pub struct ApiService;

impl ApiService {
    /// 基础 API 路径
    const API_BASE: &'static str = "/api/v1/erp";

    /// 最大重试次数
    const MAX_RETRIES: u32 = 3;

    /// GET 请求（带重试）
    pub async fn get<T: DeserializeOwned>(url: &str) -> Result<T, String> {
        Self::request_with_retry::<T>("GET", url, None).await
    }

    /// POST 请求（带重试）
    pub async fn post<T: DeserializeOwned>(url: &str, body: &serde_json::Value) -> Result<T, String> {
        Self::request_with_retry::<T>("POST", url, Some(body)).await
    }

    /// 带重试的请求方法
    async fn request_with_retry<T: DeserializeOwned>(
        method: &str,
        url: &str,
        body: Option<&serde_json::Value>,
    ) -> Result<T, String> {
        let mut last_error = None;
        let full_url = format!("{}{}", Self::API_BASE, url);

        for attempt in 0..Self::MAX_RETRIES {
            match Self::do_request(method, &full_url, body).await {
                Ok(response) => {
                    match response.json::<T>().await {
                        Ok(data) => return Ok(data),
                        Err(e) => {
                            last_error = Some(format!("解析响应失败：{}", e));
                            break; // 解析错误不重试
                        }
                    }
                }
                Err(e) => {
                    last_error = Some(e.clone());
                    
                    // 指数退避：1s, 2s, 4s
                    if attempt < Self::MAX_RETRIES - 1 {
                        let delay_ms = 1000 * 2u64.pow(attempt);
                        gloo_timers::future::TimeoutFuture::new(delay_ms as u32).await;
                    }
                }
            }
        }

        Err(last_error.unwrap_or_else(|| "未知错误".to_string()))
    }
}
```

**改进效果**:
- ✅ 提高网络请求的可靠性
- ✅ 自动处理临时网络故障
- ✅ 指数退避避免服务器压力
- ✅ 统一的错误处理机制
- ✅ 所有现有 Service 代码无需修改（接口兼容）

### 4. ✅ 添加 API 请求追踪日志

**改进文件**:
- `e:\1\10\bingxi-rust\backend\src\main.rs`

**改进内容**:
- 使用 `tower_http::trace::TraceLayer` 中间件
- 记录每个请求的方法、URI、状态码、耗时
- 记录请求失败信息

**改进代码** (`main.rs`):
```rust
use axum::{Router, http::Request};
use std::time::Duration;
use tracing::{info, warn};

// 创建路由，添加请求追踪中间件
let app = create_router(db.clone())
    .layer(
        TraceLayer::new_for_http()
            .on_request(|request: &Request<_>| {
                info!(
                    method = %request.method(),
                    uri = %request.uri(),
                    "开始处理请求"
                );
            })
            .on_response(|response: &axum::response::Response, latency: Duration| {
                info!(
                    status = %response.status(),
                    latency_ms = %latency.as_millis(),
                    "请求完成"
                );
            })
            .on_failure(|error: tower_http::BoxError| {
                warn!("请求失败：{}", error);
            })
    )
    .layer(cors)
    .layer(axum::middleware::from_fn(AuthMiddleware::new));
```

```
INFO 请求完成 status=200 latency_ms=12

```

**改进效果**:

- 日志文件命名：`bingxi_backend.log.YYYY-MM-DD`

**新增依赖** (`Cargo.toml`):
```toml

```

**改进代码** (`main.rs`):
```rust
use tracing_appender::rolling::{RollingFileAppender, Rotation};

// 使用日志轮转：每天轮转一次
let file_appender = RollingFileAppender::new(
    Rotation::DAILY,
    log_dir,
    "bingxi_backend.log",
);

tracing_subscriber::registry()
    .with(
        tracing_subscriber::EnvFilter::try_from_default_env()
            .unwrap_or_else(|_| format!("bingxi_backend={}", settings.log.level).into()),
    )
    .with(tracing_subscriber::fmt::layer()
        .with_writer(file_appender)
        .with_ansi(false)
        .with_target(true)
        .with_thread_ids(false)
        .with_file(false)
        .with_line_number(false)
    )
    .init();
```

**日志文件结构**:

```
└── ...
```

**改进效果**:
