## 二、后端错误详细分类

### 2.1 字段不存在错误 (E0560) - 约 800 个

**错误描述**: ActiveModel 或 Model 结构体中不存在指定字段

**典型错误示例**:

```
error[E0560]: struct `quality_inspection::ActiveModel` has no field named `inspection_result`
   --> src\services\quality_inspection_service.rs:173:13
    |
173 |             inspection_result: Set(req.inspection_result),
    |             ^^^^^^^^^^^^^^^^^ `quality_inspection::ActiveModel` does not have this field
    |
    = note: available fields are: `id`, `standard_name`, `standard_code`, `product_category_id`, `inspection_items` ... and 6 others
```

**影响文件**:
- `src/services/quality_inspection_service.rs` (多个错误)
- `src/services/purchase_receipt_service.rs`
- `src/services/inventory_stock_service.rs`
- 其他 Service 文件

**根本原因**:
1. 数据库表结构已变更但 Service 层代码未同步更新
2. 模型定义与服务层字段不匹配
3. 字段命名不一致

**修复建议**:
1. 检查 `src/models/` 目录下的实体定义
2. 对比数据库表结构
3. 更新 Service 层代码使用正确的字段名

---

### 2.2 枚举变体不存在 (E0599) - 约 600 个

**错误描述**: 在 Column 枚举中找不到指定的字段变体

**典型错误示例**:

```
error[E0599]: no variant or associated item named `SupplierId` found for enum `quality_inspection::Column` in the current scope
   --> src\services\quality_inspection_service.rs:248:62
    |
248 |             query = query.filter(quality_inspection::Column::SupplierId.eq(sid));
    |                                                              ^^^^^^^^^^ variant or associated item not found in `Column`
    |
   ::: src\models\quality_inspection.rs:4:35
    |
4   | #[derive(Clone, Debug, PartialEq, DeriveEntityModel, Serialize, Deserialize)]
    |                                   ----------------- variant or associated item `SupplierId` not found for this enum
```

**影响文件**:
- `src/services/quality_inspection_service.rs`
- `src/services/*.rs` (多个文件)

**根本原因**:
1. 模型中未定义该字段
2. 字段命名与代码中使用的不一致
3. SeaORM 自动生成的 Column 枚举未包含该字段

**修复建议**:
1. 在模型中添加缺失的字段
2. 或者修改查询代码使用正确的列名

---

### 2.3 方法不存在 (E0599) - 约 400 个

**错误描述**: 类型未实现指定方法

**典型错误示例**:

```
error[E0599]: `sea_orm::Select<quality_inspection::Entity>` is not an iterator
   --> src\services\quality_inspection_service.rs:304:35
    |
304 |         let total = query.clone().count(&*self.db).await?;
    |                                   ^^^^^ `sea_orm::Select<quality_inspection::Entity>` is not an iterator  
    |
help: trait `PaginatorTrait` which provides `count` is implemented but not in scope; perhaps you want to import it
    |
1   + use sea_orm::PaginatorTrait;
    |
```

**影响文件**:
- 所有使用分页查询的 Service 文件

**根本原因**:
- 缺少必要的 Trait 导入（如 `PaginatorTrait`）

**修复建议**:
在所有使用 `.count()`, `.skip()`, `.take()` 方法的 Service 文件顶部添加：
```rust
use sea_orm::PaginatorTrait;
```

---

### 2.4 类型不匹配 (E0308) - 约 300 个

**错误描述**: 表达式类型与预期类型不匹配

**典型错误示例**:
```
error[E0308]: mismatched types
  --> src/services/*.rs
   |
   |         value: RustDecimal,
   |                ^^^^^^^^^^^ expected `Decimal`, found `RustDecimal`
```

**影响文件**:
- 所有使用 Decimal 类型的模型和 Service 文件

**根本原因**:
- `RustDecimal` 和 `rust_decimal::Decimal` 类型混用

**修复建议**:
已在之前的修复中处理，统一使用 `rust_decimal::Decimal`

---

### 2.5 特征未实现 (E0277) - 约 200 个

**错误描述**: 类型未实现所需的特征

**典型错误示例**:

```
error[E0277]: the trait bound `sea_orm::Value: From<&i32>` is not satisfied
   --> src\services\quality_inspection_service.rs:297:75
    |
297 |             query = query.filter(quality_inspection::Column::ProductId.eq(product_id));
    |                                                                        -- ^^^^^^^^^^ the trait `From<&i32>` is not implemented for `sea_orm::Value`
    |                                                                        |
    |                                                                        required by a bound introduced by this call
```

**影响文件**:
- 使用引用类型进行查询过滤的 Service 文件

**根本原因**:
- 传递了引用类型而不是值类型

**修复建议**:
```rust
// 修改前
query.filter(Column::ProductId.eq(product_id))

// 修改后
query.filter(Column::ProductId.eq(*product_id))
```
