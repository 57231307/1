## 二、缺失功能详细清单

### 🔴 高优先级缺失（影响业务闭环）

#### 1. 用户管理模块 - 缺少更新和删除接口 ⚠️

**缺失接口**:
- `PUT /api/v1/erp/users/:id` - 更新用户信息
- `DELETE /api/v1/erp/users/:id` - 删除用户

**影响**:
- 用户信息无法修改（如调整部门、角色）
- 离职用户无法删除或禁用
- 用户管理功能不完整

**当前实现**:
```rust
// user_handler.rs - 只有 3 个接口
pub async fn list_users(...)   // ✅ 列表
pub async fn get_user(...)     // ✅ 详情
pub async fn create_user(...)  // ✅ 创建
// ❌ 缺少 update_user
// ❌ 缺少 delete_user
```

**建议补充代码**:
```rust
/// 更新用户
pub async fn update_user(
    State(db): State<Arc<DatabaseConnection>>,
    Path(id): Path<i32>,
    Json(req): Json<UpdateUserRequest>,
) -> impl IntoResponse {
    let user_service = UserService::new(db.clone());
    
    match user_service.update_user(
        id,
        req.email,
        req.phone,
        req.role_id,
        req.department_id,
        req.status,
    ).await {
        Ok(user) => ApiResponse::success_with_msg(user, "用户更新成功"),
        Err(e) => {
            (StatusCode::BAD_REQUEST, Json(ApiResponse::<()>::error(&format!("更新用户失败：{}", e))))
        }
    }
}

/// 删除用户
pub async fn delete_user(
    State(db): State<Arc<DatabaseConnection>>,
    Path(id): Path<i32>,
) -> impl IntoResponse {
    let user_service = UserService::new(db.clone());
    
    match user_service.delete_user(id).await {
        Ok(_) => ApiResponse::success_with_msg((), "用户删除成功"),
        Err(e) => {
            (StatusCode::BAD_REQUEST, Json(ApiResponse::<()>::error(&format!("删除用户失败：{}", e))))
        }
    }
}
```

**改进建议**: 在 `user_service.rs` 中添加对应方法，在 `routes/mod.rs` 中注册路由

#### 2. 认证模块 - 缺少注销和刷新 Token 接口 ⚠️

**缺失接口**:
- `POST /api/v1/erp/auth/logout` - 用户注销
- `POST /api/v1/erp/auth/refresh` - 刷新 Token

**影响**:
- 用户无法主动注销（只能关闭浏览器）
- Token 过期后无法刷新，需重新登录
- 用户体验不佳

**当前实现**:
```rust
// routes/mod.rs - 只有登录接口
let auth_routes = Router::new()
    .route("/login", post(auth_handler::login));
// ❌ 缺少 logout
// ❌ 缺少 refresh
```

**建议补充代码**:
```rust
/// 用户注销
pub async fn logout(
    State(db): State<Arc<DatabaseConnection>>,
    headers: HeaderMap,
) -> impl IntoResponse {
    // 获取当前用户 ID（从 Token 中）
    let user_id = get_current_user_id(&headers)?;
    
    // 可选：将 Token 加入黑名单
    // blacklist_token(token, expiry_time).await?;
    
    // 更新用户最后登录时间
    let user_service = UserService::new(db.clone());
    user_service.update_last_login(user_id, None).await?;
    
    ApiResponse::success_with_msg((), "注销成功")
}

/// 刷新 Token
pub async fn refresh_token(
    State(db): State<Arc<DatabaseConnection>>,
    headers: HeaderMap,
) -> impl IntoResponse {
    // 从旧 Token 中提取用户信息
    let claims = validate_token(&headers)?;
    
    // 生成新 Token
    let auth_service = AuthService::new(db.clone());
    let new_token = auth_service.generate_token(claims.sub, claims.username, claims.role_id)?;
    
    Ok(Json(ApiResponse::success(serde_json::json!({
        "token": new_token,
        "expires_in": 86400 // 24 小时
    }))))
}
```

**改进建议**: 补充认证相关接口，提升用户体验
