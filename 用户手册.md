## 系统简介

### 什么是面料 ERP 系统？

面料 ERP 系统是一款基于 Rust 技术栈开发的全栈式企业资源计划系统，专为面料行业设计。系统采用前后端分离架构，提供高性能、高可靠性的管理解决方案。

### 核心特性

- ✅ **高性能**: Rust 语言编写，单二进制文件部署
- ✅ **安全性**: JWT 认证 + bcrypt 密码加密 + API 限流
- ✅ **可靠性**: 数据库事务保护 + 操作日志追溯
- ✅ **易用性**: 全中文界面，操作简单直观
- ✅ **可扩展**: 模块化设计，支持功能扩展

### 技术架构

```
┌─────────────┐
│  前端界面   │ Yew + WASM
├─────────────┤
│  API 接口    │ Axum + gRPC
├─────────────┤
│  业务逻辑   │ Service 层
├─────────────┤
│  数据存储   │ PostgreSQL 18
└─────────────┘
```

## 功能模块详解

### 1. 用户管理

**功能描述**: 管理系统用户账号

**操作权限**: 系统管理员

**主要功能**:
- ✅ 创建新用户
- ✅ 查看用户列表
- ✅ 编辑用户信息
- ✅ 删除用户（软删除）
- ✅ 重置用户密码

**操作步骤**:

#### 创建用户
1. 进入【系统管理】→【用户管理】
2. 点击【新增用户】按钮
3. 填写用户信息：
   - 用户名（必填，唯一）
   - 密码（必填，至少 6 位）
   - 邮箱（可选）
   - 手机号（可选）
   - 角色（必填）
   - 部门（可选）
4. 点击【保存】按钮

#### 编辑用户
1. 在用户列表中找到目标用户
2. 点击【编辑】按钮
3. 修改用户信息
4. 点击【保存】按钮

#### 删除用户
1. 在用户列表中找到目标用户
2. 点击【删除】按钮
3. 确认删除操作

**注意事项**:
- 删除用户采用软删除，数据不会物理删除
- 删除后的用户无法恢复，需重新创建
- 用户名不能重复

### 2. 角色管理

**功能描述**: 管理系统角色和权限

**操作权限**: 系统管理员

**主要功能**:
- ✅ 创建角色
- ✅ 查看角色列表
- ✅ 编辑角色权限
- ✅ 删除角色

**权限说明**:
- `*`: 所有权限
- `module:action`: 指定模块的指定操作
  - 例如：`user:view` 表示查看用户权限
  - 例如：`product:*` 表示产品模块所有权限

**操作步骤**:

#### 创建角色
1. 进入【系统管理】→【角色管理】
2. 点击【新增角色】按钮
3. 填写角色信息：
   - 角色名称（必填，唯一）
   - 角色描述（必填）
   - 权限列表（必选）
4. 点击【保存】按钮

#### 配置权限
1. 在角色列表中找到目标角色
2. 点击【编辑】按钮
3. 勾选权限复选框
4. 点击【保存】按钮

**权限层级**:
```
一级权限：user, product, inventory, sales, finance
二级权限：view, create, update, delete, approve, reject
```

### 4. 库存管理

**功能描述**: 管理面料库存，包括库存查询、调整、盘点、调拨

**操作权限**: 具有库存模块权限的用户

#### 4.1 库存查询

**功能**: 实时查询各仓库的库存数量和金额

**查询条件**:
- 仓库筛选
- 产品筛选
- 批次筛选
- 色号筛选

**库存信息**:
- 仓库名称
- 产品信息
- 批次号
- 色号
- 当前数量
- 单位成本
- 库存金额

#### 4.2 库存调整

**功能**: 调整库存数量（增加或减少）

**调整类型**:
- `increase`: 库存增加
- `decrease`: 库存减少

**调整原因**:
- `damage`: 破损
- `sample`: 取样
- `correction`: 纠错
- `other`: 其他

**操作步骤**:
1. 进入【库存管理】→【库存调整】
2. 点击【新增调整单】按钮
3. 填写调整单信息：
   - 仓库（必填）
   - 调整日期（必填）
   - 调整类型（必填）
   - 调整原因（必填）
   - 调整明细（至少一项）
4. 选择库存记录并填写调整数量
5. 点击【保存】按钮
6. 提交审核（需要审核权限）

**审核流程**:
- 调整单创建后状态为 `pending`（待审核）
- 审核通过后状态变为 `approved`（已审核），库存数量自动更新
- 驳回后状态变为 `rejected`（已驳回），库存不变

**注意事项**:
- 减少操作时，调整数量不能大于当前库存
- 调整单一旦审核通过，不可修改
- 系统自动记录调整前后的数量变化

#### 4.3 库存盘点

**功能**: 定期盘点库存，确保账实相符

**操作步骤**:
1. 进入【库存管理】→【库存盘点】
2. 点击【新增盘点单】按钮
3. 选择盘点仓库和盘点范围
4. 录入实际盘点数量
5. 系统自动计算盘盈盘亏
6. 生成盘点报告

#### 4.4 库存调拨

**功能**: 在不同仓库之间调拨库存

**操作步骤**:
1. 进入【库存管理】→【库存调拨】
2. 点击【新增调拨单】按钮
3. 填写调拨信息：
   - 调出仓库（必填）
   - 调入仓库（必填）
   - 调拨明细
4. 确认调拨数量
5. 提交调拨申请

**注意事项**:
- 调出仓库库存必须充足
- 调拨单审核后自动更新两个仓库的库存

## 常见问题 FAQ

### 账号相关

#### Q1: 忘记密码怎么办？
**A**: 请联系系统管理员重置密码。管理员可以在【用户管理】中找到对应用户，点击【重置密码】按钮。

#### Q2: 如何修改密码？
**A**: 
1. 点击右上角用户头像
2. 选择【个人设置】
3. 输入当前密码和新密码
4. 点击【保存】按钮

#### Q3: 账号被锁定怎么办？
**A**: 连续输错密码 5 次账号会被锁定。请联系系统管理员解锁。

---

### 产品管理

#### Q4: 产品编码重复怎么办？
**A**: 产品编码必须唯一。系统会自动检测，如果重复会提示"产品编码已存在"。请修改为其他编码。

#### Q5: 如何批量修改产品价格？
**A**: 
1. 进入【产品管理】→【产品列表】
2. 勾选需要修改的产品
3. 点击【批量编辑】按钮
4. 选择"价格"字段
5. 输入新价格或调整比例
6. 点击【确认修改】按钮

#### Q6: 删除产品时提示"存在关联数据"怎么办？
**A**: 说明该产品在库存、销售等模块有使用记录。建议：
1. 先将产品状态改为"inactive"（停用）
2. 或者删除所有关联数据后再删除产品

---

### 库存管理

#### Q7: 库存调整单审核后还能修改吗？
**A**: 不能。审核后的调整单不可修改或删除。如果确需修改，需要创建一笔反向调整单进行冲销。

#### Q8: 库存数量为负数怎么办？
**A**: 系统不允许库存数量为负。减少操作时会自动检查当前库存数量。如果出现负数，说明：
1. 库存数据不准确，需要进行盘点
2. 操作顺序错误，应先入库再出库

#### Q9: 如何查询某个产品的库存变化历史？
**A**: 
1. 进入【库存管理】→【库存调整】
2. 在筛选条件中选择该产品
3. 系统会显示该产品的所有调整记录

---

### 销售管理

#### Q10: 订单审核后发现有错误怎么办？
**A**: 
1. 如果订单未发货，可以联系管理员驳回订单
2. 驳回后订单状态变为"pending"，可以修改
3. 如果订单已发货，需要创建退货单

#### Q11: 库存不足时能创建订单吗？
**A**: 可以创建，但无法审核通过。系统会在审核时检查库存数量。

#### Q12: 如何取消订单？
**A**: 
1. 只有"pending"或"approved"状态的订单可以取消
2. 在订单详情页面点击【取消订单】按钮
3. 输入取消原因
4. 确认取消

---

### 系统使用

#### Q13: 系统响应慢怎么办？
**A**: 可能原因：
1. 网络问题：检查网络连接
2. 数据量大：尝试添加筛选条件减少数据量
3. 系统负载高：联系管理员检查服务器状态

#### Q14: 页面显示异常怎么办？
**A**: 
1. 清除浏览器缓存
2. 刷新页面（Ctrl+F5）
3. 更换浏览器尝试
4. 联系管理员

#### Q15: 如何导出数据？
**A**: 
1. 在列表页面点击【导出】按钮
2. 选择导出格式（Excel/CSV）
3. 选择导出字段
4. 点击【确认导出】按钮

## 最佳实践

### 1. 账号安全

✅ **推荐做法**:
- 定期修改密码（建议每 3 个月）
- 密码复杂度：大小写字母 + 数字 + 特殊字符，至少 8 位
- 离开电脑时锁定屏幕或退出系统
- 不要共享账号

❌ **不推荐做法**:
- 使用简单密码（如 123456）
- 多个账号共用一个密码
- 将密码告诉他人
- 长时间不修改密码

---

### 2. 数据录入

✅ **推荐做法**:
- 及时录入数据，避免积压
- 录入后仔细核对
- 使用规范的命名和编码
- 填写完整的描述信息

❌ **不推荐做法**:
- 集中补录数据
- 使用模糊的描述（如"其他"）
- 编码随意编写
- 必填字段填"无"或留空

---

### 3. 库存管理

✅ **推荐做法**:
- 定期盘点（建议每月一次）
- 及时调整差异
- 设置安全库存预警
- 先进先出原则

❌ **不推荐做法**:
- 长期不盘点
- 差异长期挂账
- 库存积压严重
- 不同批次混放

---

### 4. 订单处理

✅ **推荐做法**:
- 及时审核订单
- 先审核再发货
- 异常情况及时沟通
- 完整记录订单信息

❌ **不推荐做法**:
- 订单积压不审核
- 先发货后补单
- 口头通知不录入系统
- 订单信息不完整

---

### 5. 系统性能

✅ **推荐做法**:
- 查询时添加筛选条件
- 定期清理过期数据
- 避开高峰期进行批量操作
- 定期备份数据

❌ **不推荐做法**:
- 一次性导出大量数据
- 不进行筛选查询全部数据
- 在高峰期进行批量导入
- 从不备份数据

## 故障排查

### 1. 登录问题

#### 现象：无法登录系统
**排查步骤**:
1. 检查网络连接
2. 确认服务器地址正确
3. 确认用户名和密码正确
4. 检查账号是否被锁定
5. 联系管理员确认账号状态

#### 现象：登录后自动退出
**排查步骤**:
1. 检查 Token 是否过期（默认 24 小时）
2. 重新登录
3. 清除浏览器缓存
4. 联系管理员检查会话配置

---

### 2. 数据问题

#### 现象：数据查询不到
**排查步骤**:
1. 检查筛选条件是否正确
2. 确认数据是否存在
3. 检查权限是否足够
4. 联系管理员检查数据库
