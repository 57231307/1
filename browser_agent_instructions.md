# Browser Agent 测试指令

## 任务说明

请使用Browser Agent对Bingxi ERP系统进行全面的UI功能测试。

## 测试环境

- **后端API**: http://localhost:8082
- **前端应用**: http://localhost:3000
- **测试账户**: admin / admin123 (如未初始化,需先完成系统初始化)

## 测试目标

1. 验证所有核心业务功能的UI交互正常
2. 检查表单验证和数据提交
3. 测试业务流程完整性
4. 验证权限控制
5. 测试打印功能(如已实现)
6. 发现并记录UI/UX问题

## 测试执行顺序

### 第一阶段: 认证测试 (10分钟)

**测试1: 登录功能**
```
1. 访问 http://localhost:3000/login

6. 截图保存登录成功页面
```

## 测试记录要求

对于每个测试用例,请记录:

1. **测试结果**: ✅ 通过 / ❌ 失败 / ⚠️ 部分通过
2. **截图**: 保存关键步骤的截图到 `/tmp/bingxi_browser_test_results/screenshots/`
   - 命名格式: `test_XX_description.png`
3. **问题描述**: 如果失败,详细描述问题
4. **复现步骤**: 如何复现该问题
5. **预期结果**: 应该发生什么
6. **实际结果**: 实际发生了什么

## 输出文件

测试完成后,请生成以下文件:

1. **测试报告**: `/tmp/bingxi_browser_test_results/test_report.md`
   - 包含所有测试用例的结果
   - 问题汇总
   - 性能评估
   - 改进建议

2. **截图目录**: `/tmp/bingxi_browser_test_results/screenshots/`
   - 所有测试截图

3. **问题清单**: `/tmp/bingxi_browser_test_results/issues.csv`
   ```csv
   ID,模块,页面,问题描述,严重程度,复现步骤,截图路径
   1,采购,采购订单,保存按钮点击无响应,严重,"1.打开新建页面 2.填写表单 3.点击保存",test_04_save_order.png
   ```

## 注意事项

1. **测试数据**: 使用测试数据,避免影响生产数据
2. **数据清理**: 测试结束后可以保留测试数据用于后续验证
3. **截图质量**: 确保截图清晰,能看清关键信息
4. **时间控制**: 每个测试用例控制在5分钟内
5. **异常处理**: 遇到阻塞性问题时,记录后跳过继续其他测试
