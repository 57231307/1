# P4-4 国际化（i18n）指南

> 阶段：P4 国际化
> 日期：2026-06-17
> 适用版本：bingxi-frontend 2026.1.0+

## 一、目标

为冰溪 ERP 前端集成 vue-i18n，实现中英双语切换，覆盖关键 5 页面（Dashboard / Login / Sales / Inventory / Settings）。

## 二、技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | vue-i18n@9 | Vue 3 官方推荐；Composition API 友好 |
| 模式 | Composition API（legacy: false） | 与 Pinia 风格统一 |
| 持久化 | localStorage `bingxi.locale` | 跨会话保留 |
| 默认语言 | 简体中文（zh-CN） | 业务主要市场 |
| 浏览器协商 | navigator.language | 自动检测 |

## 三、目录结构

```text
frontend/src/
├── i18n/
│   └── index.ts                # i18n 入口 + 切换 API
├── locales/
│   ├── zh-CN.ts                # 中文（简体）— 220+ 条目
│   └── en-US.ts                # English (US) — 220+ 条目
└── components/
    └── LanguageSwitcher.vue    # 语言切换组件
```

## 四、翻译覆盖（220+ 条目）

### 4.1 命名空间

```text
{module}.{section}.{key}
```

模块包括：
- `common` - 通用（30+ 条）
- `login` - 登录页（10 条）
- `dashboard` - 工作台（8 条）
- `sales` - 销售（30+ 条）
- `purchase` - 采购（10 条）
- `inventory` - 库存（30+ 条）
- `finance` - 财务（15 条）
- `settings` - 系统设置（30+ 条）
- `error` - 错误码（10 条）
- `message` - 操作消息（10 条）

### 4.2 使用示例

```vue
<template>
  <el-button>{{ $t('common.save') }}</el-button>
  <h1>{{ $t('sales.order.title') }}</h1>
  <p>{{ $t('sales.order.status.draft') }}</p>
</template>

<script setup>
import { useI18n } from 'vue-i18n'
const { t } = useI18n()
const label = t('inventory.stock.alert')
</script>
```

## 五、切换组件

`components/LanguageSwitcher.vue` 提供下拉切换：

```vue
<template>
  <LanguageSwitcher />
</template>

<script setup>
import LanguageSwitcher from '@/components/LanguageSwitcher.vue'
</script>
```

### 5.1 功能

- 显示当前语言（原生名称）
- 下拉菜单列出所有支持语言
- 当前语言 disabled 标识
- 切换后自动持久化到 localStorage
- 自动更新 `<html lang="...">` 属性（辅助屏幕阅读器）
- 自动切换 ElementPlus 内置组件语言

## 六、关键 5 页面翻译集成

| 页面 | 文件 | 关键翻译 |
|------|------|---------|
| Login | `views/Login.vue` | 标题、用户名、密码、登录按钮、错误消息 |
| Dashboard | `views/Dashboard.vue` | 欢迎语、今日订单、低库存 |
| Sales | `views/sales/` | 订单状态、客户、报价 |
| Inventory | `views/inventory/` | 库存查询、可用/锁定数量 |
| Settings | `views/Settings.vue` | 用户、角色、租户、系统配置 |

> 注：实际接入时，业务方只需把硬编码中文改为 `$t('module.key')` 即可。本 PR 集成框架 + 提供 LanguageSwitcher + 翻译字典；逐页面替换由后续业务迭代推进。

## 七、扩展新语言

1. 创建 `frontend/src/locales/{lang}.ts`，参考 `zh-CN.ts` 结构
2. 在 `i18n/index.ts` 中：
   - `SUPPORTED_LOCALES` 数组追加新语言
   - `messages` 中注册翻译
3. 测试 `LanguageSwitcher` 切换是否生效
