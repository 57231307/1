## 二、覆盖的业务规则

### 1. 销售模块（25 测试）
- 报价单状态机（6 状态）：活动/终态/可转换
- 报价单总价含税计算
- 订单类型优先级（加急/样品/常规/批量）
- 折扣类型：百分比/固定金额/阶梯
- 销售订单：最小数量、最大折扣、税额、行小计
- 退货：有效校验、退货比例、退款金额

### 2. 采购模块（20 测试）
- 采购订单状态机：可收货/已完成
- 收货完成度：完成/部分/比例
- 质检结果：合格/部分合格/不合格
- 供应商评级：D < C < B < A < S
- 应付账龄：4 段（30/60/90 天）

### 3. 库存模块（20 测试）
- 移动方向：入库（+）/出库（-）/调拨（0）
- 库存预警：低库存/超库存/正常
- 盘点差异：平衡/盘盈/盘亏
- 调整原因：6 种（盘点/报损/过期/丢失/盘盈/系统）
- 库存计算：合计、入库/出库应用

### 4. AR 模块（15 测试）
- 应收单状态：已收款/逾期
- 付款方式：5 种（现金/银行/承兑/支付宝/微信）
- 应收单计算：余额、已付比例、逾期判定
- 对账：差额、对账率、部分对账

### 5. BI 模块（20 测试）
- 时间粒度：日/周/月/季/年
- 趋势方向：上升/下降/平稳
- 客户分级：D < C < B < A < VIP
- 销售汇总：客单价、人均
- ABC 分类
- 财务指标：毛利、毛利率、净利、净利率、盈利判定

## 三、技术细节

### 单元测试特点
- **零 DB 依赖**：所有测试为纯函数测试，使用 rust_decimal 进行精确计算
- **中文友好**：使用 `dec!` 宏 + 业务术语（应收/应付/毛利/账龄）
- **边界覆盖**：每个核心函数都包含：正常值/边界值/异常值
- **不依赖时间**：通过 `crate::ymd!` 宏显式传入测试日期

### 测试组织
- 5 个独立 `#[cfg(test)]` 模块
- 每个模块只引入必要依赖（rust_decimal、chrono）
- 每个 test 函数命名：`test_XX_业务场景_期望结果`
