> **版本**: v1.0

> **目标用户**: 销售员 / 工艺员 / 质检员 / 售后客服

## 1. 模块概述

定制订单全流程跟踪模块（Custom Order Full Process Tracking）用于管理客户特殊定制订单从下单到交付售后的完整生命周期。

### 1.1 业务背景

纺织行业的客户经常需要定制特殊规格的面料（指定纱线、染色方法、后整理工艺等）。本模块覆盖：

- 销售员：创建和管理定制订单
- 工艺员：跟踪 5 阶段工艺流程
- 质检员：上报和处理质量异常
- 售后客服：处理客诉、维修、换货、退款

### 1.2 5 阶段工艺流程

```
草稿 (draft)
  ↓ 销售员创建 / 从销售订单转
纱线采购 (yarn_purchasing)
  ↓ 采购员操作
染整 (dyeing)
  ↓ 染色车间
后整理 (finishing)
  ↓ 后整理车间
交付 (delivery)
  ↓ 物流签收
售后 (after_sales)
  ↓ 售后处理
已完成 (completed)
```

## 2. 操作指南

### 2.1 创建定制订单

**路径**: `/custom-orders/new`

**步骤**：
1. 填写客户 ID、产品 ID
2. 选择色号（可选）
3. 输入规格（如：100% 棉 200g/m² 幅宽 1.5m）
4. 输入数量 + 选择单位（米/千克/件）
5. 选择纱线规格、染色方法、后整理方法（可选）
6. 选择期望交付日期
7. 输入总金额 + 选择币种
8. 关联销售订单 ID（可选，从销售订单转定制订单时使用）
9. 填写定制要求备注
10. 点击"保存草稿"

**系统自动行为**：
- 生成订单号（格式：CO + YYYYMMDD + 4 位序号）
- 自动创建 5 阶段工艺节点（纱线采购/染整/后整理/交付/售后）

### 2.2 跟踪工艺流程

**路径**: `/custom-orders/:id/track`

**功能**：
- 5 阶段甘特图（已计划/实际时间）
- 节点状态颜色（蓝=进行中/绿=已完成/红=阻塞/灰=待开始）
- 完整操作日志时间线
- 节点操作按钮（开始/暂停/完成/阻塞）

### 2.3 推进订单状态

**操作**：详情页 → "推进状态"按钮

**自动行为**：
- 当前节点标记为"已完成"（记录 actual_end_date）
- 下一节点标记为"进行中"（记录 actual_start_date + operator_id）
- 记录工艺日志（before_status → after_status）

### 2.4 上报质量异常

**路径**: 详情页 → 质量异常 Tab → "上报异常"

**填写**：
- 异常类型：
  - 色差（GB/T 26377）— 可选填 ΔE 值
  - 色牢度（ISO 105）— 可选填 1-5 等级
  - 规格不符 / 破损 / 其他
- 严重度：低 / 中 / 高 / 严重
- 描述（必填）

**行业规则**：
- ΔE > 1.0 提示可感知色差
- ΔE > 5.0 警告超过行业阈值
- ISO 105 色牢度等级 1-5 校验

### 2.5 处理售后工单

**路径**: 详情页 → 售后 Tab → "创建售后工单"

**4 种类型**：
- 客诉（complaint）：记录客户不满
- 维修（repair）：返厂维修
- 换货（exchange）：换新品
- 退款（refund）：必须填写退款金额

**状态机**：
- opened（已开） → processing（处理中） → resolved（已解决）→ closed（已关闭）
- 可从 opened 直接 rejected（拒绝）
- 不可从 closed 回到其他状态

## 3. 状态机说明

### 3.1 订单级状态机

| 当前状态 | 允许推进到 |
|---------|----------|
| draft（草稿） | yarn_purchasing（纱线采购） |
| yarn_purchasing | dyeing（染整） |
| dyeing | finishing（后整理） |
| finishing | delivery（交付） |
| delivery | after_sales（售后） |
| after_sales | completed（已完成） |
| 任意非终态 | cancelled（已取消） |
| completed / cancelled | 终态，不可推进 |

### 3.2 工艺节点状态机

| 当前状态 | 允许操作 |
|---------|---------|
| pending（待开始） | start（开始） |
| in_progress（进行中） | pause（暂停）/ complete（完成）/ block（阻塞） |
| blocked（阻塞） | unblock（解除阻塞） |
| completed（已完成） | 终态 |

### 3.3 售后状态机

| 当前状态 | 允许转换到 |
|---------|----------|
| opened | processing / rejected / closed |
| processing | resolved / closed / rejected |
| resolved | closed |
| closed / rejected | 终态 |

## 4. 行业规则

### 4.1 GB/T 26377-2022 纺织品颜色标准

- **色差 ΔE 阈值**：
  - ΔE ≤ 1.0：不可感知色差
  - 1.0 < ΔE ≤ 2.0：轻微色差
  - 2.0 < ΔE ≤ 3.5：明显色差
  - ΔE > 3.5：严重色差

### 4.2 ISO 105 色牢度

- **耐洗色牢度**（ISO 105-C06）
- **耐光色牢度**（ISO 105-B02）
- **耐摩擦色牢度**（ISO 105-X12）
- **等级**：1（最差）→ 5（最优）

## 5. 常见问题

### Q1: 如何从销售订单转定制订单？

A: 销售订单详情页有"转定制订单"按钮（需 `product.is_customizable=true`），点击后自动创建草稿。

### Q2: 草稿状态以外可以修改吗？

A: 不可以。订单推进到纱线采购阶段后，只能推进状态或取消，不能修改基础信息。如需修改请联系管理员。

### Q3: 节点阻塞后如何恢复？

A: 详情页工艺节点卡片 → "解除阻塞" 按钮 → 节点恢复到 in_progress 状态。

### Q4: 退款类型工单为什么必须填金额？

A: 业务规则要求：退款工单必须有具体退款金额用于财务核销。

### Q5: 多租户隔离如何生效？

A: 所有 API 调用自动从 JWT token 提取 tenant_id，仅返回当前租户数据。跨租户访问返回 403。
