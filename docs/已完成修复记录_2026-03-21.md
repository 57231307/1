## 修复 3：match 臂类型不匹配

**错误代码**: E0308  
**错误信息**: ``match` arms have incompatible types`

**问题描述**: Ok 分支和 Err 分支返回的 `ApiResponse` 泛型类型不一致

**修复前**:
```rust
match product_service.create_product(...).await {
    Ok(product) => {
        (StatusCode::CREATED, Json(ApiResponse::success_with_msg(product, "产品创建成功")))
    }
    Err(e) => {
        (StatusCode::BAD_REQUEST, Json(ApiResponse::<()>::error(&format!("创建产品失败：{}", e))))
        //                      ^^^^^^^^^^^^^^^^ 类型不一致
    }
}
```

**修复后**:
```rust
match product_service.create_product(...).await {
    Ok(product) => {
        (StatusCode::CREATED, Json(ApiResponse::success_with_msg(product, "产品创建成功")))
    }
    Err(e) => {
        (StatusCode::BAD_REQUEST, Json(ApiResponse::error(&format!("创建产品失败：{}", e))))
        //                      ^^^^^^^^^^^^^ 类型推断，保持一致
    }
}
```

**修复说明**: 移除 `ApiResponse::<()>::` 中的显式类型注解 `::<()>`，让编译器自动推断类型。

**影响的方法**:
- `create_product`
- `update_product`
- `get_product`
- `delete_product`
- `list_product_colors`
- `create_product_color`
- `update_product_color`
- `delete_product_color`
- `batch_create_colors`

**修复说明**: `batch_create_product_colors` 方法期望的参数类型是 `Vec<(String, String, Option<String>, String, Option<String>, f64)>`（tuple 类型），而非 `CreateProductColorInput` struct。
