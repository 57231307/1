# 纺织面料 ERP：双单位换算系统设计规范 (Dual Unit Converter)

## 1. 业务痛点与系统目标
在面料行业，产品的生产和内部仓储通常使用“米 (m)”或“公斤 (kg)”，但客户下单或报关时可能需要“码 (yd)”或“磅 (lb)”。更为复杂的是，由于面料的克重（g/m²）和门幅（cm）不同，长度（米）与重量（公斤）的转换比例并不是一个定值。

本模块旨在建立一个**自动、精准、无误差的全局双单位换算引擎**，使得业务员在前端开单时只需输入客户需要的单位，系统便能在底层自动换算并正确扣减库存。

## 2. 核心架构设计 (根据用户决策)

### 2.1 底层存储模式：强主单位模式 (Master Unit)
- **绝对核心**: 在库存表（Inventory）中，所有涉及面料长度的物理存储**强制统一使用“米 (m)”**。
- **业务解耦**: 前端开单（采购单、销售单）允许选择任何辅单位（码、公斤、磅）。当单据生效并执行库存出入库动作时，后端拦截器会自动根据公式将其转换回“米”存入数据库。
- **优势**: 杜绝了“同一匹布记两笔账”导致的对账灾难。

### 2.2 换算公式层级：产品级绑定 (Product-Level Binding)
换算公式分为两大类：
1. **同维度的全局固定换算 (长度转长度 / 重量转重量)**:
   - 1 码 (yd) = 0.9144 米 (m)
   - 1 磅 (lb) = 0.453592 公斤 (kg)
   - *这些属于物理定律，系统内置为全局常量，不需要绑定产品。*
2. **跨维度的产品级换算 (长度转重量)**:
   - 由于每种面料（如 200g/m² 的棉布和 100g/m² 的涤纶）单位米数的重量不同，因此“1 米 = X 公斤”的公式必须直接**挂靠在产品档案 (Product)** 上。
   - 在前端创建/编辑产品时，需新增两个字段：`门幅 (width, cm)` 和 `克重 (weight, g/m²)`。系统根据公式自动推算出该产品的“米/公斤换算系数”，并允许人工微调覆盖。

### 2.3 精度控制：底层高精度，界面显示 2 位
- **后端数据库 (Sea-ORM)**: 涉及换算后的米数/公斤数，统一采用 `DECIMAL(14, 4)` 数据类型（即保留 4 位小数参与底层加减乘除计算），彻底避免“100笔四舍五入后丢了2米布”的累计误差问题。
- **前端显示 (Yew)**: 在数据表格、卡片和打印单据中，使用 `format!("{:.2}", value)` 将其格式化为 2 位小数显示给用户。

## 3. UI/UX 交互设计

### 3.1 产品档案界面的改造
在“产品管理 -> 新建/编辑产品”的表单中，新增一块“规格与换算”的区域（保持 Compact UI 的紧凑布局）：
- **门幅 (cm)**: [ input ]
- **克重 (g/m²)**: [ input ]
- **换算系数**: [ 1米 = 0.0000 公斤 ] *(输入门幅和克重后，系统通过 JavaScript 实时计算并展示在这个只读框中)*

### 3.2 换算规则管理页面 (`DualUnitConverter` 页面)
这是一个位于基础数据下的独立页面，用于集中管理和查看所有的单位关系。
- **上半部分**: 展示全局常量（如 米转码 的系数），只读或仅限超级管理员修改。
- **下半部分**: 一个紧凑的、带搜索功能的列表，展示目前系统中所有面料产品的跨维度换算系数一览表（包含产品编号、名称、门幅、克重、1米对应公斤数）。支持管理员在这里一键批量修改换算系数。

## 4. API 与后端数据流设计
1. **`GET /api/v1/erp/base/units/constants`**: 获取全局固定换算率（如码到米）。
2. **`GET /api/v1/erp/base/units/products`**: 获取所有产品的米/公斤换算列表。
3. **`PUT /api/v1/erp/base/units/products/{id}`**: 手动微调某个产品的换算系数。

*(注意：在实施计划中，我们将优先完成基础的后端骨架和前端页面的展示)*
