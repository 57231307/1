# 面料行业 ERP 前端重构设计规范 (Compact UI)

## 1. 背景与目标
当前秉羲 ERP 系统的前端界面（基于 Rust + Yew + CSS）默认采用了较为宽松的 padding 和字体设计，这无法满足纺织面料行业多维度、高密度数据展示的诉求。
本设计规范旨在对系统进行全局的“紧凑模式”改造，并将缺失的纺织行业专属功能模块补充到二级、三级侧边栏菜单中。

## 2. 界面设计规范 (CSS 改造)
**目标文件**: `/workspace/frontend/styles/main.css`

### 2.1 紧凑型全局变量
新增一套专为高密度 ERP 设计的 CSS 变量：
- `--sidebar-width`: 由 `250px` 压缩至 `200px`。
- `--spacing-compact`: 表格内边距缩小至 `4px 8px`。
- `--font-size-base`: 由 `14px` 或 `16px` 缩小至 `13px`。
- `--font-size-small`: 设定为 `12px`。

### 2.2 表格与数据展示 (Table & Data)
- **行高限制**: 表格行高限制为 `32px`，极大提升同屏可视数据量。
- **等宽数字**: 新增 `.numeric-cell` 类，强制应用 `font-variant-numeric: tabular-nums;`，并设置 `text-align: right;`，便于业务员核对金额、米数、重量等数据。
- **微型状态标签**: 缩小 `.status-badge` 的尺寸（如 `padding: 2px 6px`），使用实心圆点或更清爽的背景色。

## 3. 导航与侧边栏重构 (Navigation)
**目标文件**: `/workspace/frontend/src/components/navigation.rs`

### 3.1 结构重构
- 现有的扁平列表需改造为**手风琴式多级折叠菜单 (Accordion)**。
- 引入 `yew::use_state` 记录当前展开的菜单组（如记录某个字符串或 ID）。
- 菜单层级分为：一级分类（带图标）、二级页面模块。

### 3.2 纺织行业专属菜单树规划
需在导航组件中补齐完整的二/三级功能列表（包括尚未实现的占位页面）：
1. **工作台**: 仪表板
2. **基础数据**: 产品档案, 产品分类, 仓库管理, 部门管理, 角色管理, 用户管理, 供应商, 客户管理, **双单位换算**
3. **销售与CRM**: 销售订单, **面料订单(新增)**, 销售合同, 销售退货, 销售价格, 客户信用, 销售分析, CRM线索, CRM商机
4. **生产管理(核心)**: **坯布管理, 染化料配方, 染缸批次, 面料批次, 质检管理**
5. **库存管理**: 库存查询, **库存盘点, 库存调拨, 库存调整**
6. **采购管理**: 采购订单, 采购合同, 采购收货, 采购退货, 采购价格, **入厂检验, 供应商评价**
7. **财务与成本**: 应收账款, 应付账款, 收款核销, 付款核销, 资金管理, 财务分析, 凭证管理, 预算管理, **成本收集, 辅助核算, 业务追溯**, 固定资产

## 4. 实施策略与限制
- **无破坏性重构**: 所有修改均在 UI 层，不得更改后端 API 路由或破坏现有的 `Route` 逻辑。尚未实现的路由统一指向现有的页面（如果存在）或新增枚举占位。
- **Yew 组件兼容**: 使用 Rust/Yew 的标准宏语法 `html! { ... }` 编写。

## 5. 验收标准
- 侧边栏宽度变窄，且支持点击折叠/展开子菜单。
- 业务表格行高明显变小，数字列强制右对齐。
- 全局无编译警告或错误，`cargo check` 正常通过。
