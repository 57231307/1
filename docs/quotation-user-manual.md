# 报价单用户手册

> **模块路径**：销售管理 → 报价单管理
> **适用版本**：冰溪 ERP 2026.1+
> **最后更新**：2026-06-16

报价单是销售流程的核心单据，本模块为面料行业定制，支持 **多币种 + 阶梯价 + Incoterms 2020 + 客户等级折扣 + 4 类贸易条款**。

## 1. 核心概念

### 1.1 报价单状态

| 状态 | 中文 | 说明 |
|------|------|------|
| `draft` | 草稿 | 可编辑、可删除 |
| `pending_approval` | 待审批 | 等待审批人处理 |
| `approved` | 已批准 | 可转销售订单 |
| `rejected` | 已拒绝 | 可重新编辑提交 |
| `expired` | 已过期 | 超过有效期自动变为此状态 |
| `converted` | 已转订单 | 已生成销售订单 |
| `cancelled` | 已取消 | 手工取消，无法恢复 |

### 1.2 价格条款（Incoterms 2020）

| 代码 | 全称 | 中文 | 适用场景 |
|------|------|------|----------|
| `FOB` | Free On Board | 装运港船上交货 | 最常用的海运条款 |
| `CIF` | Cost, Insurance and Freight | 成本+保险+运费 | 含海运保险 |
| `EXW` | Ex Works | 工厂交货 | 国内短途/买方自提 |
| `DDP` | Delivered Duty Paid | 完税后交货 | 卖方承担所有费用 |
| `DAP` | Delivered At Place | 目的地交货 | 不含关税 |

### 1.3 报价单号规则

`QT + YYYYMMDD + 4 位序号`，例如 `QT202606160001` 表示 2026-06-16 的第 1 张报价单。

## 2. 创建报价单

### 操作步骤

1. 进入 **销售管理 → 报价单管理** 页面
2. 点击右上角 **新建报价单** 按钮
3. 填写表单：
   - **客户**：从客户下拉框选择（仅显示已启用的客户）
   - **报价日期**：默认今天
   - **有效期至**：默认 30 天后
   - **价格条款**：5 种 Incoterms 2020 必选其一
   - **币种**：CNY / USD / EUR
   - **汇率**：自动锁定（CNY = 1.0，外币需手动调整）
   - **含税**：默认开
   - **客户等级**：VIP / NORMAL（影响折扣）
4. **添加报价明细**：
   - 点击「添加产品」按钮
   - 选择产品（自动加载该产品的色号下拉）
   - 选择色号（可选）
   - 输入数量和单价
   - 含税单价自动 = 单价 × 1.13
5. **添加贸易条款**（4 个 Tab 自由组合）
6. 点击 **保存草稿** 或 **提交审批**

### 字段说明

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 客户 | 必填 | 必须已启用 |
| 报价日期 | 必填 | YYYY-MM-DD |
| 有效期至 | 必填 | 必须 > 报价日期 |
| 价格条款 | 必填 | 5 种 Incoterms 之一 |
| 币种 | 必填 | CNY / USD / EUR |
| 汇率 | 必填 | > 0 |
| 含税 | 可选 | 默认 true |
| 客户等级 | 可选 | VIP 享 95 折 |
| MOQ | 可选 | 最小起订量 |
| 交期(天) | 可选 | 交货周期 |
| 备注 | 可选 | 任意文本 |
| 至少 1 个产品明细 | 必填 | 产品 + 数量 > 0 |

## 3. 审批流程

### 3.1 金额阶梯审批

| 金额 | 审批人 | 说明 |
|------|--------|------|
| **< 10 万** | 销售员自批 | 提交后直接「已批准」 |
| **10 - 50 万** | 销售经理 | 提交后状态变「待审批」 |
| **> 50 万** | 总经理 | 提交后状态变「待审批」 |

### 3.2 提交审批

1. 在草稿状态点击 **提交审批** 按钮
2. 系统自动判断金额区间
3. < 10 万自动批准；其他进入 BPM 流程
4. 即使 BPM 模板未配置，提交也能成功

### 3.3 审批操作（在「待审批」状态可见）

| 角色 | 可执行操作 |
|------|------------|
| 提交人 | 撤回（实际为取消后重新编辑） |
| 审批人 | 批准 / 拒绝（必须填拒绝原因） |
| 任何有权限用户 | 查看审批进度 |

## 4. 报价单详情与操作

进入报价单详情页（列表点击「查看」或 URL `/quotations/:id`），可执行：

### 4.1 顶部按钮（按状态显示）

| 状态 | 可用按钮 |
|------|----------|
| 草稿 / 已拒绝 | 返回、编辑、保存、提交审批、取消 |
| 待审批 | 返回、批准、拒绝 |
| 已批准 | 返回、转销售订单、取消 |
| 已过期 / 已转订单 / 已取消 | 仅返回（只读） |

### 4.2 详情页内容

- **基本信息**：客户、日期、价格条款、币种、客户等级、状态
- **报价明细表**：产品、色号、规格、单位、数量、单价、含税单价、金额
- **贸易条款**：4 个 Tab 分别展示
- **金额合计**：小计、税额、合计
- **元数据**：创建时间、更新时间

## 5. 贸易条款

报价单支持 4 类贸易条款，每类可添加多条：

| 类型 | 中文 | 典型内容 |
|------|------|----------|
| `logistics` | 物流条款 | 运输方式、装运港、目的港、交货时间 |
| `payment` | 付款条件 | 付款方式、账期、币种、汇率锁定 |
| `sample` | 样品条款 | 样品费、快递费、是否抵扣货款 |
| `inspection` | 检验条款 | 检验标准、检验地点、第三方机构 |

贸易条款是**非结构化文本**，但支持类型分组展示。

## 6. 转销售订单

### 6.1 前置条件

- 报价单状态 = `approved`（已批准）
- 报价单未过期

### 6.2 操作步骤

1. 在「已批准」报价单详情页点击 **转销售订单** 按钮
2. 二次确认（弹窗提示转订单后状态不可恢复）
3. 系统自动：
   - 创建销售订单草稿（订单号格式 `SO + YYYYMMDD + 4 位`）
   - 复制所有报价明细
   - 复制客户、价格、币种、汇率
   - 备注自动添加 `[源自报价单 XXXX]` 前缀
4. 报价单状态变为「已转订单」
5. 跳转到新销售订单详情页

### 6.3 注意事项

- 转订单是**不可逆**操作
- 一个报价单只能转一次
- 销售订单生成后，可独立编辑、提交、审批

## 7. 失效处理

### 7.1 自动过期

- 超过 `valid_until` 的「已批准」报价单，下次访问时自动标记为「已过期」
- 过期后不能转订单，需重新创建报价单

### 7.2 取消

- 在「草稿 / 待审批 / 已拒绝 / 已批准」状态下可手动取消
- 取消后状态变为「已取消」，不可恢复
- 取消理由可在备注中记录

### 7.3 即将过期提醒

- 后端提供 `/quotations/expiring` 端点，返回 7 天内到期的报价单
- 前端可在仪表盘展示提醒卡片

## 8. 常见问题

### Q1: 提交审批后多久能批下来？

A: 取决于审批人。可在审批页查看审批进度，状态实时更新。

### Q2: 报价单可以编辑吗？

A: 草稿和已拒绝状态可编辑。其他状态需先取消或重新创建。

### Q3: 为什么提示「色号价格未配置」？

A: 该产品的色号价格表未维护，需先在产品管理中配置色号价格。

### Q4: 阶梯价怎么生效？

A: 阶梯价在 `product_color_prices` 表中按 `min_quantity` 配置，系统自动匹配。

### Q5: VIP 客户折扣自动应用吗？

A: 是的。报价时选择「客户等级 = VIP」后，提交审批前的预计算会自动应用 95 折。

### Q6: 报价单能转为采购单吗？

A: 暂不支持。只能转为销售订单。

### Q7: 如何批量导入报价单？

A: 暂不支持批量导入。可通过 API 单条创建，详见 `docs/quotation-api.md`。

### Q8: 客户等级和客户档案中的等级不一致时以谁为准？

A: 以报价单上的 `customer_level` 字段为准。如未指定，默认 NORMAL。

### Q9: 贸易条款是否必填？

A: 否。所有贸易条款都是可选的，根据实际业务需要添加。

### Q10: 转订单后能修改销售订单的价格吗？

A: 可以。销售订单生成后是独立草稿，可由销售员自行调整。
