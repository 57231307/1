## 二、P1 问题清单（18 项，按修复优先级排序）

### P1-1：mark_as_paid 审计 user_id 仍为 Some(0)（批次 59b 遗漏项）
- **文件**：`backend/src/services/ar_invoice_service.rs:416`
- **问题**：`mark_as_paid` 函数签名 `pub async fn mark_as_paid(&self, id: i32)` 不接收 user_id，审计日志用 `Some(0)` 占位
- **对比**：同文件 `cancel`/`approve`/`update` 均已透传 user_id
- **修复**：函数签名改为 `mark_as_paid(&self, id: i32, user_id: i32)`，调用方 ar_collection_service.confirm 透传

### P1-2：22 处 Json<serde_json::Value> 无类型校验
- **文件**：21 个 handler 文件（budget_management_handler / customer_credit_handler / financial_analysis_handler / sales_order_handler / login_security_handler / production_order_handler / crm_handler / crm_customer_handler / purchase_inspection_handler / fixed_asset_handler / supplier_evaluation_handler / five_dimension_handler / purchase_contract_handler / report_enhanced_handler / sales_contract_handler）
- **问题**：手动解析 Json<Value>，无字段长度/范围/必填校验
- **修复**：参照 finance_invoice_handler.rs 的 CreateFinanceInvoiceDto 模式批量改造

### P1-3：report 模块 4 处 .expect("合法时分秒") panic 风险
- **文件**：`backend/src/services/report/ds.rs:260, 264` + `backend/src/services/report/job.rs:70, 113`
- **问题**：生产代码 expect 在 None 时 panic
- **修复**：改为 `ok_or_else(|| AppError::internal("时分秒非法"))?`

### P1-4：3 处 let _ = 静默吞错
- **文件**：
  - `backend/src/services/failover_service.rs:233, 246`（熔断状态写入失败静默丢弃）

  - `backend/src/services/bpm_service.rs:829`（通知创建失败静默丢弃）
- **修复**：改为 `if let Err(e) = ... { tracing::warn!("失败: {}", e); }`

### P1-5：finance_invoice_handler 金额字段 f64 → Decimal
- **文件**：`backend/src/handlers/finance_invoice_handler.rs:45, 48, 51`
