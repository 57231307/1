## 一、模块介绍

色卡仓储管理是面向纺织行业色卡全生命周期的管理模块，支持：

- 色卡创建、查询、归档
- 色号批量导入（CSV / Excel）
- 色卡借出、归还、遗失、损坏登记
- 二维码扫码查询色号详情
- 色号颜色空间转换（RGB / CMYK / CIELab）
- 色号导出 CSV
- 多租户隔离

## 二、操作指南

### 2.1 创建色卡

1. 登录冰溪 ERP 系统
2. 进入「色卡仓储管理 → 色卡列表」
3. 点击「新建色卡」按钮
4. 填写色卡基本信息：
   - **色卡编号**（必填，唯一）：例如 `PANTONE-TPX-2024-SS`
   - **色卡名称**（必填）：例如 `2024 春夏 PANTONE 色卡`
   - **色卡类型**（必填）：PANTONE / CNCS / 自定义
   - **季节**：可选，例如 `2024SS`
   - **品牌**：可选
   - **描述**：可选
5. 点击「立即创建」

成功后会显示色卡 ID，可以立即点击「添加色号」进入详情页。

### 2.2 添加色号（单个）

1. 进入色卡详情页
2. 切换到「色号列表」Tab
3. 点击「添加色号」按钮
4. 填写色号信息：
   - **色号编码**（必填）：例如 `18-1664 TPX`
   - **色号名称**（必填）：例如 `番茄红`
   - **颜色选择**：使用色块选择器，RGB / HEX 实时联动
   - **PANTONE / CNCS 编码**：可选
5. 点击「添加」

系统会自动：
- 同步更新 HEX 和 RGB
- 自动计算 CMYK 印刷色彩空间
- 自动计算 CIELab 色彩空间
- 更新色卡总色号数

### 2.3 批量导入色号

#### 方式 A：通过 UI（CSV 文本输入）

1. 进入色卡详情页 → 「色号列表」Tab
2. 点击「批量导入」按钮
3. 在文本框中按行输入色号，格式：
   ```
   18-1664 TPX,番茄红,220,50,50,#DC3232
   17-1463 TPX,柑橘橙,250,130,50,#FA8232
   ```
4. 字段顺序：`color_code, color_name, rgb_r, rgb_g, rgb_b, hex_value`
5. 点击「导入」

#### 方式 B：通过 API（适合大批量）

```bash
curl -X POST http://server/api/v1/erp/color-cards/{id}/items/batch \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d @items.json
```

**性能**：1000 条 < 5 秒

### 2.4 借出色卡

1. 进入「色卡借出管理」页面
2. 切换到「借出色卡」Tab
3. 选择色卡（下拉搜索）
4. 输入客户 ID
5. 选择预计归还时间（可选）
6. 填写用途（如：客户选色 / 展会展示）
7. 填写备注（可选）
8. 点击「确认借出」

**业务规则**：
- 预计归还时间不能晚于借出时间 + 30 天
- 仅 `active` 状态的色卡可借出

### 2.5 归还色卡

1. 在「借出管理」页切换到「借出中」Tab
2. 找到对应借出记录
3. 点击「归还」按钮
4. 填写实际归还时间 + 备注
5. 确认归还

### 2.6 登记遗失（含赔付）

1. 在「借出管理」页切换到「借出中」Tab
2. 找到对应借出记录
3. 点击「遗失」按钮
4. 填写赔付金额（必须 > 0）
5. 填写遗失原因
6. 确认登记

**联动效果**：
- 借出记录状态变为 `lost`
- 色卡状态自动变为 `lost`
- 写入审计日志

### 2.7 标记损坏

类似遗失流程，赔付金额可选（可填 0 表示无需赔付）。

### 2.8 扫码查询色号

1. 在「色号列表」中点击任意色号的「扫码」按钮，或
2. 直接访问 `GET /api/v1/erp/color-cards/scan/{color_code}`

返回：
- 色号完整信息（RGB / CMYK / Lab）
- 所属色卡信息
- 关联染色配方（如有）
- 关联色号价格（如有）

### 2.9 导出色卡为 CSV

1. 进入色卡详情页
2. 点击「导出 CSV」按钮
3. 浏览器下载 `color-card-{card_no}.csv` 文件

CSV 包含所有色号及完整色彩空间数据。

## 三、常见问题

### Q1: 色卡编号已存在？
A: 色卡编号全局唯一，请使用新的编号或在原色卡上添加色号。

### Q2: 批量导入失败？
A: 检查：
- color_code 是否在同一色卡内重复
- rgb_r/g/b 是否在 0-255
- hex_value 是否为 `#RRGGBB` 格式

### Q3: 借出失败？
A: 检查：
- 色卡状态是否为 `active`
- 预计归还时间是否在 30 天内

### Q4: 遗失后还能恢复吗？
A: 不能。遗失是终态操作。如需恢复，请联系管理员在数据库中手动修复。

### Q5: 扫码查询色号没有配方信息？
A: 配方是可选关联。如需关联，请在添加色号时填写 `dye_recipe_id`。

### Q6: CIELab 值如何理解？
A:
- **L**：亮度（0 黑 → 100 白）
- **a**：红绿轴（-128 绿 → +127 红）
- **b**：黄蓝轴（-128 蓝 → +127 黄）
- 行业标准：ΔE 色差 ≤ 3 视为合格

## 四、最佳实践

### 4.1 色卡编号规范
建议格式：`{类型}-{细分}-{季节}`
- `PANTONE-TPX-2024-SS`
- `CNCS-S1050-2024-AW`
- `CUSTOM-BX-C-001`

### 4.2 色号命名规范
- 中文名称简洁明了：`番茄红` / `柑橘橙` / `深海蓝`
- 同时填写 PANTONE / CNCS 编码方便跨体系对照
- 多个客户共用色号时使用通用自定义编码（如 `BX-C-001`）

### 4.3 借出管理建议
- 借出时**必填**用途
- 预计归还时间不超过 30 天
- 借出超过 14 天未归还时主动跟进

### 4.4 批量导入性能
- 每次导入不超过 1000 条
- 大批量（> 1000 条）分批导入
- 导入失败时查看 `errors` 字段修正后重新导入失败的色号

### 4.5 色彩空间使用
- **RGB**：用于屏幕显示（设计稿、屏幕预览）
- **CMYK**：用于印刷（包装、吊牌、宣传册）
- **CIELab**：用于色差判定（打样确认、来料检验）

## 五、权限要求

| 操作 | 所需权限 |
|------|----------|
| 查看色卡列表 / 详情 | 基础权限 |
| 创建 / 更新 / 归档色卡 | 色卡管理权限 |
| 添加 / 更新 / 删除色号 | 色号管理权限 |
| 批量导入色号 | 色号管理权限 + 导入权限 |
| 借出 / 归还 | 借出管理权限 |
| 登记遗失 / 损坏 | 借出管理权限 + 财务确认 |
| 扫码查询 | 基础权限 |
| 导出 CSV | 色卡管理权限 |
