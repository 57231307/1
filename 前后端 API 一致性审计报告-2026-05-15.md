## 一、严重问题（🔴）

### 问题 1: 通知 API 路径错误

**位置**: `/workspace/frontend/src/api/notification.ts`

**问题描述**:
```typescript
// 前端 (错误)
return request.get('/notifications', { params })

// 后端 (正确)
.nest("/api/v1/erp/notifications", Router::new()
    .route("/", get(notification_handler::list_notifications))
```

**影响**:
- 所有通知相关请求将返回 404 错误
- 通知功能完全无法使用

**修复方案**:
```typescript
// 方案 1: 修改所有端点添加 /erp 前缀
return request.get('/erp/notifications', { params })

// 方案 2 (推荐): 修改 request.ts baseURL
baseURL: '/api/v1/erp'  // 保持不变，但 notification.ts 需要使用完整路径
```

**严重程度**: 🔴 高
