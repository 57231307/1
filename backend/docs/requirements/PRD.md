## 1. 产品概述

### 1.1 产品定位
秉羲ERP是一款面向面料纺织行业的综合性企业资源计划系统，涵盖从采购、生产、库存、销售到财务的全业务流程管理。

### 1.2 目标用户
- 面料纺织企业管理人员
- 采购/销售/仓库/财务人员
- 生产计划人员
- 质量检验人员

### 1.3 技术架构
- 后端: Rust + Axum 0.7 + SeaORM 0.12
- 数据库: PostgreSQL
- 前端: Yew (Rust WASM)
- 部署: 支持单机/容器化部署

## 3. 缺失功能需求 (待开发)

### 3.1 高优先级缺失 (P1)

#### REQ-001: 生产计划管理 (MRP)
**需求描述**: 基于销售订单和库存情况，自动生成生产计划和物料需求计划
**业务价值**: 面料行业核心需求，减少人工计算错误
**功能点**:
- 生产订单CRUD
- 物料需求计算(BOM展开)
- 产能负荷分析
- 生产排程甘特图
- 缺料预警

#### REQ-002: 数据外键关联统一
**需求描述**: BPM/CRM/ERP核心实体间建立数据库外键关联
**业务价值**: 保证数据一致性，消除数据孤岛
**功能点**:
- BPM流程实例添加business_type枚举约束
- CRM线索与客户建立外键关联
- 成本归集与生产批次建立外键
- 质量检验与采购入库建立外键

#### REQ-003: 权限检查补全
**需求描述**: 所有敏感操作添加权限验证
**业务价值**: 防止越权访问，提升系统安全性
**功能点**:
- 审查所有Handler的权限检查
- 添加缺失的permission中间件
- 敏感操作添加操作日志

### 3.2 中优先级缺失 (P2)

#### REQ-004: 应收对账模块
**需求描述**: 与客户进行应收账款对账
**业务价值**: 完善财务闭环，减少坏账风险
**功能点**:
- 应收对账单生成
- 客户对账确认/争议
- 账龄分析
- 自动对账

#### REQ-005: 多币种支持
**需求描述**: 支持多种货币的业务处理
**业务价值**: 满足外贸业务需求
**功能点**:
- 币种管理
- 汇率管理
- 多币种订单/发票
- 汇率自动更新

#### REQ-006: 单元测试覆盖
**需求描述**: 核心Service层单元测试
**业务价值**: 提升代码质量，减少回归bug
**功能点**:
- 核心Service测试用例
- 集成测试
- 测试覆盖率报告

#### REQ-007: 代码文档补全
**需求描述**: pub函数添加文档注释
**业务价值**: 提升代码可维护性
**功能点**:
- 所有pub函数添加rustdoc
- 模块级文档
- 示例代码

### 3.3 低优先级缺失 (P3)

#### REQ-008: AI智能分析
**需求描述**: 基于历史数据的智能预测和推荐
**业务价值**: 辅助决策，提升运营效率
**功能点**:
- 销售预测
- 库存优化建议
- 异常检测
- 智能推荐

#### REQ-009: 多租户支持
**需求描述**: SaaS化多租户部署
**业务价值**: 支持云服务模式
**功能点**:

#### REQ-010: 插件机制

## 4. 非功能需求

### 4.1 性能需求
- API响应时间 < 200ms (P95)
- 并发用户支持 > 1000
- 数据库连接池 max_connections = 100

### 4.2 安全需求
- 密码强度: 8位以上，包含大小写+数字+特殊字符
- JWT Token过期: 2小时
- 敏感操作需要二次确认
- 审计日志完整记录

### 4.3 可用性需求
- 系统可用性 > 99.9%
- 支持 graceful shutdown
- 健康检查端点

### 4.4 兼容性需求
- 数据库: PostgreSQL 14+
- 浏览器: Chrome/Firefox/Edge 最新版
- 移动端: 响应式布局

## 5. 数据需求

### 5.1 数据关联关系

```
客户(customer) --1:N--> 销售订单(sales_order)
客户(customer) --1:N--> 应收发票(ar_invoice)
客户(customer) --1:N--> CRM商机(crm_opportunity)

供应商(supplier) --1:N--> 采购订单(purchase_order)
供应商(supplier) --1:N--> 应付发票(ap_invoice)
供应商(supplier) --1:N--> 供应商评估(supplier_evaluation)

产品(product) --1:N--> 库存(inventory_stock)
产品(product) --1:N--> 批次(batch)

BPM流程实例(bpm_process_instance) --N:1--> BPM流程定义(bpm_process_definition)
BPM任务(bpm_task) --N:1--> BPM流程实例(bpm_process_instance)
```

### 5.2 数据孤岛改进

| 关联关系 | 当前状态 | 改进方案 | 优先级 |
|----------|----------|----------|--------|
| BPM-业务 | 字符串关联 | 添加外键约束 | P1 |
| CRM-ERP客户 | 无关联 | 添加customer_id外键 | P1 |
| 成本-生产 | 字符串关联 | 添加batch_id外键 | P2 |
| 质量-采购 | 弱关联 | 添加receipt_id外键 | P2 |
