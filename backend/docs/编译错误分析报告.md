### 2.2 方法未找到错误 (E0599)

**错误位置**: 多个 service 文件

**错误示例 1 - 缺少 PaginatorTrait**:
```
error[E0599]: no method named `count` found for struct `sea_orm::Select` in the current scope
   --> src\services\financial_analysis_service.rs:60:35
    |
60  |         let total = query.clone().count(&*self.db).await?;
    |                                   ^^^^^
    |
help: trait `PaginatorTrait` which provides `count` is implemented but not in scope
    |
1   + use sea_orm::PaginatorTrait;
```

**错误示例 2 - 缺少 QuerySelect**:
```
error[E0599]: no method named `offset` found for struct `sea_orm::Select` in the current scope
   --> src\services\financial_analysis_service.rs:69:14
    |
69  |             .offset(params.page * params.page_size)
    |             -^^^^^^ method not found in `Select<Entity>`
    |
help: trait `QuerySelect` which provides `offset` is implemented but not in scope
    |
1   + use sea_orm::QuerySelect;
```

**错误原因**:
- SeaORM 的查询方法分布在不同的 trait 中
- `count()` 方法需要 `PaginatorTrait`
- `offset()` 和 `limit()` 方法需要 `QuerySelect`
- 这些 trait 没有被导入到使用它们的作用域

**涉及文件**:
- `src/services/financial_analysis_service.rs`
- `src/services/purchase_price_service.rs`
- `src/services/sales_price_service.rs`
- `src/services/sales_analysis_service.rs`
- `src/services/quality_inspection_service.rs`

**解决方案**:
在每个需要分页查询的 service 文件顶部添加：
```rust
use sea_orm::{EntityTrait, QueryFilter, ColumnTrait, QuerySelect, PaginatorTrait};
```

### 2.3 类型转换错误 (E0277)

**错误位置**: 
- `src/services/purchase_price_service.rs:36,40`
- `src/services/sales_price_service.rs:36`

**错误示例**:
```
error[E0277]: the trait bound `sea_orm::Value: From<&i32>` is not satisfied
  --> src\services\purchase_price_service.rs:36:71
   |
36 |             query = query.filter(purchase_price::Column::ProductId.eq(product_id));
   |                                                                    -- ^^^^^^^^^^
   |                                                                    |
   |                                                                    required by a bound introduced by this call
   |
   = note: required for `&i32` to implement `Into<sea_orm::Value>`
```

**错误原因**:
- `product_id` 和 `supplier_id` 参数类型是 `&i32`（引用类型）
- SeaORM 的 `eq()` 方法需要所有权值，不支持引用
- 编译器无法自动将 `&i32` 转换为 `sea_orm::Value`

**解决方案**:
使用解引用操作符 `*`:
```rust
// 修改前
query = query.filter(purchase_price::Column::ProductId.eq(product_id));

// 修改后
query = query.filter(purchase_price::Column::ProductId.eq(*product_id));
```

**错误示例**:
```
error[E0252]: the name `Utc` is defined multiple times
 --> src\models\assist_accounting.rs:1:14
  |
1 | use chrono::{Utc, DateTime};
  |              ^^^ `Utc` reimported here
```

**错误原因**:

**解决方案**:
- 合并重复的 use 语句
- 或者使用 `use crate_name::module::Item;` 的完整路径

### 2.5 列变体未找到错误

**错误位置**: `src/services/financial_analysis_service.rs:149,150`

**错误示例**:
```
error[E0599]: no variant or associated item named `IndicatorId` found for enum `financial_analysis::Column`
   --> src\services\financial_analysis_service.rs:149:49
    |
149 |             .filter(financial_analysis::Column::IndicatorId.eq(indicator_id))

    |
149 |             .filter(financial_analysis::Column::IndicatorCode.eq(indicator_id))
```

**错误原因**:
- Service 代码使用了错误的列名

**解决方案**:

```rust

// 修改后
.filter(financial_analysis::Column::IndicatorCode.eq(indicator_id))
```

## 三、错误根源分析

### 3.1 主要原因

1. **Model 与 Service 不同步**
   - 数据库表结构变更后，Model 定义已更新
   - 但 Service 层代码仍在使用旧的字段名
   - 缺少自动化的代码生成或同步机制

2. **缺少必要的 Trait 导入**
   - SeaORM 使用 trait 来组织查询功能
   - 开发者可能不熟悉 SeaORM 的架构
   - 代码复制粘贴时遗漏了 use 语句

3. **类型系统不匹配**
   - Rust 的强类型系统要求精确的类型匹配
   - 引用和所有权的混淆
   - 缺少类型转换的显式处理

### 3.2 次要原因

1. **代码审查不足**
   - 新增功能时没有充分测试
   - 缺少自动化类型检查流程

2. **文档不完整**
   - SeaORM 的 trait 系统文档不够清晰
   - 团队内部缺少最佳实践分享
