## 二、安全性审计（严重问题）

### 2.1 [必须修复] 认证中间件被绕过

**位置**: [auth.rs:21-30](file:///workspace/backend/src/middleware/auth.rs#L21-L30)

**问题描述**: 认证中间件中存在临时调试代码，直接返回管理员身份，完全绕过了 JWT 验证。

```rust
// 当前代码（严重漏洞）
pub async fn auth_middleware(...) -> Result<Response, StatusCode> {
    // TEMPORARY DEBUG BYPASS: 插入管理员AuthContext以确保权限中间件通过
    let auth_context = AuthContext {
        user_id: 0,
        username: "admin".to_string(),
        role_id: Some(1),
    };
    request.extensions_mut().insert(auth_context);
    return Ok(next.run(request).await);  // 直接返回，跳过所有认证！
    // ... 后续代码永远不会执行
}
```

**影响**: 
- 任何用户无需登录即可访问所有接口
- 所有请求都以管理员身份执行
- 权限控制完全失效

**修复建议**: 删除第 21-30 行的调试代码，恢复正常的 JWT 验证流程。

pub async fn anti_brute_force(...) -> Result<Response, AppError> {

**修复建议**: 删除第 96-98 行的调试代码，恢复正常的速率限制。

### 2.4 [必须修复] 配置文件包含明文敏感信息

**位置**: [config.yaml:8-13](file:///workspace/backend/config.yaml#L8-L13)

**问题描述**: 配置文件中包含明文数据库密码和 JWT 密钥。

```yaml
database:
  connection_string: "postgres://bingxi:d5eb610ccf1a701dac02d5.dbcba8f5f546a@39.99.34.194:5432/bingxi"
  password: "d5eb610ccf1a701dac02d5.dbcba8f5f546a"

auth:
  jwt_secret: "local-dev-secret-key-for-test-environments-only-32-bytes"
  cookie_secret: "local-dev-cookie-secret-key-must-be-at-least-32-bytes-long"
```

**影响**:
- 数据库密码泄露风险
- JWT 密钥泄露可导致任意用户身份伪造

**修复建议**:
1. 将敏感配置移至环境变量
2. 使用 `.env` 文件并添加到 `.gitignore`
3. 生产环境使用密钥管理服务
