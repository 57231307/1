# ERP 系统问题修复规划

## 问题总结

### 1. 品牌名称问题（已修复 ✅）
**问题**：系统界面和文档中使用了"秉羲"品牌名称，需要移除

**修复**：
- 已全局搜索并替换所有"秉羲"字样
- 统一使用"面料管理"或"ERP"命名
- 修改 52 个文件（前端、后端、文档、部署脚本等）

**验证**：
- 前端登录页面显示"面料管理系统"
- 侧边栏标题显示"面料管理"
- 所有文档更新完成

### 2. CI/CD 时区问题（已修复 ✅）
**问题**：CI/CD 流水线生成版本号时使用 UTC 时区，与中国时间不一致

**修复**：
- 在 `.github/workflows/ci-cd.yml` 中添加 `export TZ="Asia/Shanghai"`
- 版本号生成逻辑使用中国时区时间

**验证**：
- 下次构建将自动生成正确的中国时间版本号

### 3. CLI 工具不可用问题（已修复 ✅）
**问题**：SSH 中使用 `bingxi` 命令无法执行

**根因**：
- `install.sh` 的 `update` 函数中未调用 `setup_cli`
- 更新后 CLI 工具可能不是最新版本

**修复**：
- 在 `update` 函数中添加 `setup_cli` 调用
- 确保更新操作也会安装最新的 CLI 工具

**验证**：
- 新安装：`setup_cli` 在 `install` 函数中被调用 ✅
- 更新操作：`setup_cli` 现在也会被调用 ✅
- CLI 路径：`/usr/local/bin/bingxi`（在 PATH 中）✅

### 4. 后端模型字段不匹配问题（已修复 ✅）
**问题**：后端模型字段与数据库实际列名不匹配，导致 109 个编译错误

**修复**：
- 修复 finance_payment 模型（移除 9 个不存在字段）
- 修复 finance_invoice 模型（移除 4 个不存在字段）
- 修复 product_category 模型（移除 3 个不存在字段）
- 修复 purchase_inspection 模型（添加 2 个缺失字段）
- 修复 purchase_return 模型（添加 3 个缺失字段）
- 修复 ar_reconciliation 模型（更新字段名）
- 修复 currency、exchange_rate、cost_collection 模型
- 修复所有相关服务和处理器文件

**验证**：
- 编译错误从 109 个减少到 0 个
- 仅剩警告（未使用的代码）
