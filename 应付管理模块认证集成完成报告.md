## 🔧 技术实现

### 1. 认证中间件

**文件**: `backend/src/middleware/auth_context.rs`

```rust
/// 用户认证信息
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AuthContext {
    /// 用户 ID
    pub user_id: i32,
    /// 用户名
    pub username: String,
    /// 角色 ID
    pub role_id: Option<i32>,
}

impl<S> FromRequestParts<S> for AuthContext
where
    S: Send + Sync,
{
    type Rejection = (StatusCode, &'static str);

    async fn from_request_parts(parts: &mut Parts, _state: &S) -> Result<Self, Self::Rejection> {
        // 从扩展中获取 Claims（由 auth middleware 插入）
        if let Some(claims) = parts.extensions.get::<Claims>() {
            Ok(AuthContext::from(claims.clone()))
        } else {
            Err((StatusCode::UNAUTHORIZED, "未认证"))
        }
    }
}
```

**核心功能**:
- 自动从 JWT Token 中提取用户信息
- 通过 Axum Extractor 模式注入 Handler
- 未认证请求自动返回 401 错误

### 2. 日志框架

**使用的 crate**: `tracing`

**日志级别**:
- `info!`: 正常操作记录（查询、创建、更新、删除等）
- `warn!`: 验证错误、业务警告
- `error!`: 系统错误、数据库错误

**日志格式**:
```rust
// 查询操作
info!("用户 {} 查询付款申请列表，供应商 ID: {:?}", auth.username, params.supplier_id);
info!("用户 {} 查询付款申请成功，共 {} 条记录", auth.username, total);

// 创建操作
info!("用户 {} 创建付款申请，供应商 ID: {}", auth.username, req.supplier_id);
warn!("用户 {} 创建付款申请验证失败：{}", auth.username, e);
info!("用户 {} 创建付款申请成功：{}", auth.username, request.request_no);

// 错误处理
error!("用户 {} 操作失败：{}", auth.username, error_message);
```

### 3. Handler 更新模式

**标准模式**（所有 Handler 统一遵循）:

```rust
pub async fn create_request(
    Json(req): Json<CreateApPaymentRequest>,
    State(db): State<Arc<DatabaseConnection>>,
    auth: AuthContext,  // ← 新增认证参数
) -> Result<Json<ApiResponse<serde_json::Value>>, AppError> {
    // ← 新增操作日志
    info!("用户 {} 创建付款申请，供应商 ID: {}", auth.username, req.supplier_id);
    
    // 验证错误日志
    req.validate().map_err(|e| {
        warn!("用户 {} 创建付款申请验证失败：{}", auth.username, e);  // ← 新增警告日志
        AppError::ValidationError(e.to_string())
    })?;
    
    let service = ApPaymentRequestService::new(db);
    // 使用 auth.user_id 替代硬编码的 user_id = 1
    let request = service.create(req, auth.user_id).await?;  // ← 使用认证用户 ID
    
    // 成功日志
    info!("用户 {} 创建付款申请成功：{}", auth.username, request.request_no);  // ← 新增成功日志
    
    Ok(Json(ApiResponse::success_with_message(
        "付款申请创建成功",
        serde_json::to_value(request)?,
    )))
}
```

## 🔐 安全性提升

### 1. 用户身份追溯

**改进前**:
```rust
let user_id = 1; // 硬编码，无法追溯实际用户
```

**改进后**:
```rust
// 从 JWT Token 自动提取，无法伪造
info!("用户 {} (ID: {}) 执行操作", auth.username, auth.user_id);
```

### 2. 操作审计

所有关键操作现在都有完整日志记录：
- **Who**: 哪个用户（username, user_id）
- **When**: 操作时间（日志时间戳）
- **What**: 执行了什么操作（创建、更新、删除等）
- **Which**: 操作了哪个对象（单据 ID、供应商 ID 等）
- **Result**: 操作结果（成功/失败，失败原因）

### 3. 权限控制基础

虽然本次任务主要是认证集成，但为后续权限控制打下基础：
- 已获取用户角色 ID（`auth.role_id`）
- 可在 Service 层根据角色进行权限判断
- 可實現数据级权限（用户只能访问本部门数据）

## 🎓 开发规范总结

### 1. Handler 层标准模式

```rust
// 1. 导入认证上下文和日志
use crate::middleware::auth_context::AuthContext;
use tracing::{info, warn, error};

// 2. Handler 函数签名添加 auth 参数
pub async fn xxx_handler(
    // ... 其他参数
    auth: AuthContext,  // ← 必须放在最后
) -> Result<..., AppError> {
    
    // 3. 记录操作开始
    info!("用户 {} 执行操作...", auth.username);
    
    // 4. 验证错误记录警告
    req.validate().map_err(|e| {
        warn!("用户 {} 验证失败：{}", auth.username, e);
        AppError::ValidationError(e.to_string())
    })?;
    
    // 5. 使用 auth.user_id
    let result = service.xxx(..., auth.user_id).await?;
    
    // 6. 记录操作成功
    info!("用户 {} 操作成功", auth.username);
    
    Ok(...)
}
```

### 2. 日志级别使用规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| `info!` | 正常操作、查询、业务成功 | "用户 xxx 查询成功" |
| `warn!` | 验证失败、业务警告 | "用户 xxx 验证失败：xxx" |
| `error!` | 系统错误、数据库错误 | "用户 xxx 数据库操作失败：xxx" |

### 3. 日志内容规范

**必须包含的信息**:
- 用户名：`auth.username`
- 操作类型：查询/创建/更新/删除/审批等
- 关键业务参数：供应商 ID、单据号、金额等
- 操作结果：成功/失败、记录数量等

**推荐格式**:
```rust
// 查询
info!("用户 {} 查询 XXX 列表，条件：{}", auth.username, conditions);
info!("用户 {} 查询成功，共 {} 条", auth.username, count);

// 创建
info!("用户 {} 创建 XXX，关键参数：{}", auth.username, key_params);
info!("用户 {} 创建成功：{}", auth.username, no);

// 错误
warn!("用户 {} 操作失败：{}", auth.username, error_reason);
```
