# 秉羲 ERP 系统 - 部署指南

## 部署概述

**版本**: 2026-03-15  
**技术栈**: Rust 全栈（后端 Axum + 前端 Yew）  
**数据库**: PostgreSQL 18  
**部署方式**: 二进制文件部署

---

## 一、环境要求

### 1.1 服务器要求

**操作系统**: 
- Linux (推荐 Ubuntu 20.04+)
- Windows Server 2019+

**硬件配置**:
- CPU: 2 核心及以上
- 内存：4GB 及以上
- 磁盘：20GB 及以上

**软件依赖**:
- PostgreSQL 18
- Nginx（前端静态文件托管）
- systemd（Linux 进程守护）

---

## 二、数据库部署

### 2.1 安装 PostgreSQL 18

**Ubuntu**:
```bash
# 添加 PostgreSQL 仓库
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt/ $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

# 安装 PostgreSQL 18
sudo apt update
sudo apt install postgresql-18 postgresql-contrib-18
```

**Windows**:
下载安装包：https://www.postgresql.org/download/windows/

---

### 2.2 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -i -u postgres

# 进入 psql
psql

# 创建数据库
CREATE DATABASE bingxi_erp OWNER postgres;

# 创建用户（请修改密码）
CREATE USER bingxi_user WITH PASSWORD 'your_secure_password';

# 授权
GRANT ALL PRIVILEGES ON DATABASE bingxi_erp TO bingxi_user;

# 退出
\q
exit
```

---

### 2.3 执行数据库迁移

```bash
# 复制迁移脚本
cp backend/database/migration/001_init.sql /tmp/

# 执行迁移
psql -U bingxi_user -d bingxi_erp -f /tmp/001_init.sql
```

## 三、后端部署

### 3.1 编译后端

```bash
# 进入后端目录
cd backend

# 编译 Release 版本
cargo build --release

# 编译完成后，二进制文件位于：
# target/release/bingxi_backend
```

---

### 3.2 配置后端

**创建配置文件** `.env`:
```ini
# 运行环境
ENV=production

# 服务器配置
SERVER__HOST=0.0.0.0
SERVER__PORT=8080

# 数据库配置
DATABASE__CONNECTION_STRING=postgres://bingxi_user:your_secure_password@localhost:5432/bingxi_erp

# JWT 配置（请修改为安全密钥）
AUTH__JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
AUTH__TOKEN_EXPIRY_HOURS=24

# 日志配置
LOG__LEVEL=info
LOG__DIR=/var/log/bingxi_erp/
```

**重要**: 
- 生产环境必须修改 `JWT_SECRET`
- 数据库密码使用强密码
- 日志目录需要提前创建

---

### 3.3 创建 systemd 服务（Linux）

**创建服务文件** `/etc/systemd/system/bingxi-backend.service`:
```ini
[Unit]
Description=Bingxi ERP Backend Service
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/bingxi/backend
ExecStart=/opt/bingxi/backend/bingxi_backend
Restart=on-failure
RestartSec=5

# 环境变量
Environment="RUST_LOG=info"

# 日志
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

**启动服务**:
```bash
# 重载 systemd 配置
sudo systemctl daemon-reload

# 启用服务
sudo systemctl enable bingxi-backend

# 启动服务
sudo systemctl start bingxi-backend

# 查看状态
sudo systemctl status bingxi-backend

# 查看日志
sudo journalctl -u bingxi-backend -f
```

### 4.2 配置 Nginx

**创建 Nginx 配置文件** `/etc/nginx/sites-available/bingxi-erp`:
```nginx
server {
    listen 80;
    server_name your-domain.com;  # 修改为实际域名

    # 前端静态文件
    location / {
        root /opt/bingxi/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket 支持（如果需要）
    location /ws {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
    }

    # 日志
    access_log /var/log/nginx/bingxi_erp_access.log;
    error_log /var/log/nginx/bingxi_erp_error.log;
}
```

**启用站点**:
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/bingxi-erp /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

## 五、防火墙配置

### 5.1 Linux (UFW)

```bash
# 允许 HTTP
sudo ufw allow 80/tcp

# 允许 HTTPS（如果配置了 SSL）
sudo ufw allow 443/tcp

# 允许 PostgreSQL（仅内网）
sudo ufw allow from 192.168.1.0/24 to any port 5432

# 启用防火墙
sudo ufw enable
```

### 5.2 Windows Firewall

```powershell
# 允许 HTTP
New-NetFirewallRule -DisplayName "Bingxi ERP HTTP" -Direction Inbound -LocalPort 80 -Protocol TCP -Action Allow

# 允许后端
New-NetFirewallRule -DisplayName "Bingxi Backend" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Allow
```

### 6.2 配置 HTTPS

修改 Nginx 配置：
```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # ... 其他配置
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 7.2 数据库备份

**创建备份脚本** `/opt/bingxi/scripts/backup.sh`:
```bash
#!/bin/bash

BACKUP_DIR="/opt/bingxi/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="bingxi_erp"
DB_USER="bingxi_user"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -U $DB_USER $DB_NAME | gzip > $BACKUP_DIR/$DB_NAME_$DATE.sql.gz

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/$DB_NAME_$DATE.sql.gz"
```

**添加到 crontab**:
```bash
# 每天凌晨 2 点备份
0 2 * * * /opt/bingxi/scripts/backup.sh
```

### 7.3 健康检查

**创建健康检查脚本** `/opt/bingxi/scripts/health_check.sh`:
```bash
#!/bin/bash

# 检查后端服务
if systemctl is-active --quiet bingxi-backend; then
    echo "✓ Backend service is running"
else
    echo "✗ Backend service is NOT running"
    exit 1
fi

# 检查 Nginx
if systemctl is-active --quiet nginx; then
    echo "✓ Nginx is running"
else
    echo "✗ Nginx is NOT running"
    exit 1
fi

# 检查 PostgreSQL
if systemctl is-active --quiet postgresql; then
    echo "✓ PostgreSQL is running"
else
    echo "✗ PostgreSQL is NOT running"
    exit 1
fi

# 检查 HTTP 响应
HTTP_CODE=$(curl -o /dev/null -s -w "%{http_code}\n" http://localhost/api/v1/erp/dashboard/overview)
if [ "$HTTP_CODE" -eq 200 ] || [ "$HTTP_CODE" -eq 401 ]; then
    echo "✓ HTTP endpoint is responding (code: $HTTP_CODE)"
else
    echo "✗ HTTP endpoint is NOT responding (code: $HTTP_CODE)"
    exit 1
fi

echo "All health checks passed!"
```

## 九、性能优化

### 9.1 数据库优化

**添加索引**（已包含在迁移脚本中）:
```sql
-- 检查索引
SELECT tablename, indexname, indexdef
FROM pg_indexes
WHERE schemaname = 'public'
ORDER BY tablename, indexname;
```

**优化配置** `/etc/postgresql/18/main/postgresql.conf`:
```conf
# 内存配置（根据服务器配置调整）
shared_buffers = 256MB
effective_cache_size = 1GB
work_mem = 16MB
maintenance_work_mem = 128MB

# 连接配置
max_connections = 100

# 日志配置
log_min_duration_statement = 1000  # 记录超过 1 秒的查询
```

## 十一、更新和升级

### 11.1 后端更新

```bash

sudo systemctl stop bingxi-backend

# 备份旧版本
```

```bash

cd /opt/bingxi/frontend
trunk build --release

# 备份旧版本
cp -r /opt/bingxi/frontend/dist /opt/bingxi/frontend/dist.bak

# 替换静态文件
rm -rf /opt/bingxi/frontend/dist
cp -r /opt/bingxi/frontend/dist /opt/bingxi/frontend/

```
