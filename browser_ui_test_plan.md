**测试目标**: 使用内置浏览器代理对所有前端页面进行完整的UI功能测试  
**测试范围**: 58个前端页面,覆盖采购、销售、库存、财务、基础数据等所有业务模块

## 🎯 执行策略

### 自动化测试脚本

使用Browser Agent依次执行以下测试序列:

```javascript
// 伪代码示例
const testSuite = [
  { name: "登录测试", page: "/login", actions: [...] },
  { name: "采购订单测试", page: "/purchase-orders", actions: [...] },
  // ... 更多测试
];

for (const test of testSuite) {
  await runTest(test);
  await captureScreenshot();
  await generateReport();
}
```

### 并行测试

对于独立的模块测试,可以启动多个Browser Agent并行执行:
- Agent 1: 采购模块测试
- Agent 2: 销售模块测试
- Agent 3: 库存模块测试
- Agent 4: 财务模块测试

## ⚠️ 注意事项

1. **测试数据隔离**: 每个测试用例应使用独立的测试数据,避免相互影响
2. **数据清理**: 测试结束后清理测试数据,保持数据库干净
3. **截图保存**: 每个失败的测试都应保存截图以便分析
4. **日志记录**: 详细记录每个操作步骤和结果
5. **超时处理**: 设置合理的超时时间,避免无限等待
