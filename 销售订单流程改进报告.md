# 销售订单流程改进报告

## 改进日期
2026-03-15

## 改进目标
修复销售订单流程中缺失的库存管理功能，实现完整的订单流转。

---

## 改进前的问题

### 问题 1: 创建订单不检查库存 ❌
**影响**: 可以创建库存不足的订单，导致无法发货

### 问题 2: 审核订单不锁定库存 ❌
**影响**: 可能导致超卖（一货多卖）

### 问题 3: 发货不扣减库存 ❌
**影响**: 订单已发货但库存未变，账实不符

---

## 改进内容

### 1. 添加库存检查功能 ✅

**文件**: `backend/src/services/sales_service.rs`

**新增方法**:
```rust
/// 检查库存是否充足
async fn check_inventory(
    &self,
    items: &[CreateSalesOrderItemRequest],
    txn: &sea_orm::DatabaseTransaction,
) -> Result<(), sea_orm::DbErr>
```

**功能说明**:
- 遍历订单明细项
- 查询每个产品的库存数量
- 检查库存是否满足订单需求
- 库存不足时返回详细错误信息

**集成位置**: `create_order()` 方法中

**代码示例**:
```rust
// 创建订单主表后，检查库存
self.check_inventory(&request.items, &txn).await?;
```

### 4. 实现订单审核功能 ✅

**新增方法**:
```rust
pub async fn approve_order(&self, order_id: i32) -> Result<SalesOrderDetail, sea_orm::DbErr>
```

**审核流程**:
1. 检查订单是否存在
2. 检查订单状态（必须为 draft）
3. 开启事务
4. 锁定库存
5. 更新订单状态为 approved
6. 记录审核人和审核时间
7. 提交事务
8. 返回订单详情

**状态流转**: `draft` → `approved`

## 完整的订单流转流程

```
┌─────────────┐
│  创建订单   │ draft 状态
│  ✓ 检查库存 │
│  ✓ 计算金额 │
│  ✓ 生成订单 │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审核订单   │ approved 状态
│  ✓ 锁定库存 │
│  ✓ 记录审核 │
│  ✓ 更新状态 │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  发货处理   │ shipped 状态
│  ✓ 扣减库存 │
│  ✓ 记录发货 │
│  ✓ 更新状态 │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  收货确认   │ completed 状态
│  ✓ 完成订单 │
│  ✓ 更新状态 │
└─────────────┘
```

---

## 状态机定义

| 状态 | 说明 | 允许的操作 | 下一状态 |
|------|------|-----------|---------|
| draft | 草稿 | 编辑、删除、审核 | approved |
| approved | 已审核 | 发货、取消 | shipped |
| shipped | 已发货 | 完成、退货 | completed |
| completed | 已完成 | 无 | - |

**建议**: 创建 `inventory_reservations` 表
**作用**: 记录订单锁定库存的详细信息
**字段**:
```sql
CREATE TABLE inventory_reservations (
    id SERIAL PRIMARY KEY,
    order_id INTEGER NOT NULL,
    product_id INTEGER NOT NULL,
    warehouse_id INTEGER NOT NULL,
    quantity DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    reserved_at TIMESTAMPTZ DEFAULT NOW(),

    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
