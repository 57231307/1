| M018 | 库存预留 | 库存预留、释放 | `inventory_reservations` |

### 1.2 技术栈确认

- **后端框架**: Axum + Tokio (HTTP 服务)
- **ORM 框架**: SeaORM 1.0 + PostgreSQL 18
- **通信协议**: Tonic (gRPC 模块间通信)
- **前端框架**: Yew + Trunk (Rust 全栈)
- **部署方式**: 后端二进制 + Nginx 静态文件

### 1.3 面料行业特色

- **批次管理**: 面料生产批次追踪（染色批次、织造批次）
- **色号管理**: 面料颜色管理（色号、色差等级）
- **等级管理**: 面料品级（一等品、二等品、等外品）
- **米数/重量双计量**: 面料按米数和克重双单位管理
- **缸号管理**: 染色缸号追踪（同缸色差控制）

## 二、47 个缺失模块详细分析

### 2.1 财务管理模块（15 个）

#### M019 - 总账管理
**优先级**: P0 (核心)  
**依赖关系**: 无依赖  
**核心功能**:
- 会计科目管理（科目树、科目余额）
- 记账凭证（凭证录入、审核、过账）
- 账簿管理（总账、明细账、日记账）
- 财务报表（资产负债表、利润表、现金流量表）

**数据库表**:
- `finance_account_subjects` - 会计科目
- `finance_vouchers` - 记账凭证
- `finance_voucher_items` - 凭证分录
- `finance_general_ledger` - 总账
- `finance_subsidiary_ledger` - 明细账

**与现有模块集成**:
- 与 `finance_payments` 集成：收款单自动生成凭证
- 与 `finance_invoices` 集成：发票自动生成凭证
- 与 `sales_orders` 集成：销售确认生成凭证

## 四、与现有 18 个模块的集成方案

### 4.1 集成原则

1. **数据一致性**: 通过 SeaORM 事务保证跨模块数据一致性
2. **接口统一性**: 所有接口遵循 `/api/v1/erp/{模块}/{操作}` 格式
3. **事件驱动**: 关键业务操作触发事件，其他模块订阅处理
4. **gRPC 通信**: 模块间通信使用 Tonic gRPC

### 4.2 关键集成点

#### 4.2.1 财务模块集成

**现有模块**: `finance_payments`, `finance_invoices`

**集成方案**:
- M019 总账：收款/付款/发票操作自动生成记账凭证
- M020 应收：销售订单确认时生成应收，收款时核销
- M021 应付：采购订单确认时生成应付，付款时核销
- M029 现金：收款/付款自动更新现金日记账

**集成示例**:
```rust
// 创建收款单时触发
#[transaction]
async fn create_payment(db: &DatabaseConnection, payment_data: PaymentData) -> Result<()> {
    // 1. 创建收款单
    let payment = finance_payment::ActiveModel::insert(payment_data.clone()).exec(db).await?;
    
    // 2. 生成记账凭证
    let voucher = VoucherData {
        type: "收款凭证".to_string(),
        items: vec![
            VoucherItem { subject: "银行存款", debit: payment.amount, credit: Decimal::ZERO },
            VoucherItem { subject: "应收账款", debit: Decimal::ZERO, credit: payment.amount },
        ],
    };
    finance_voucher::ActiveModel::insert(voucher).exec(db).await?;
    
    // 3. 核销应收
    if let Some(receivable_id) = payment.receivable_id {
        write_off_receivable(db, receivable_id, payment.amount).await?;
    }
    
    Ok(())
}
```

#### 4.2.2 供应链模块集成

**现有模块**: `sales_orders`, `inventory_stock`, `customers`

**集成方案**:
- M034 采购：采购订单入库时更新 `inventory_stock`
- M040 销售报价：报价转订单时创建 `sales_orders`
- M042 销售发货：发货时扣减 `inventory_stock.quantity_available`
- M045 信用：下单前检查客户信用额度

**集成示例**:
```rust
// 销售发货扣减库存
async fn ship_order(db: &DatabaseConnection, order_id: i32) -> Result<()> {
    let order = SalesOrder::find_by_id(order_id).one(db).await?;
    
    for item in &order.items {
        // 扣减库存
        let stock = InventoryStock::find_by_id(item.warehouse_id, item.product_id)
            .one(db).await?;
        
        let mut stock_active = stock.into_active_model();
        stock_active.quantity_on_hand -= item.quantity;
        stock_active.quantity_available -= item.quantity;
        stock_active.last_movement_date = Some(Utc::now());
        stock_active.update(db).await?;
    }
    
    Ok(())
}
```

## 五、数据库设计规范

### 5.1 命名规范

**表命名**:
- 格式：`{模块}_{业务对象}`
- 示例：`finance_vouchers`, `purchase_orders`, `production_orders`

**字段命名**:
- 主键：`id` (i32)
- 外键：`{关联表}_id` (i32)
- 金额：使用 `Decimal` 类型
- 日期：使用 `DateTime<Utc>`
- 状态：使用 `String` (active/inactive, pending/approved/rejected)

**中文注释**:
```rust
/// 会计科目表

pub struct Model {

    pub id: i32,
    /// 科目编码（唯一）
    pub code: String,
    /// 科目名称
    pub name: String,
    // ...
}
```

### 5.2 索引规范

**必须建索引的字段**:
- 外键字段
- 高频查询字段（如订单号、产品编码）
- 排序字段（如创建时间、状态）
