## 一、功能概述

### 1.1 模块定位
采购管理模块是面料 ERP 系统的核心供应链模块之一，负责管理从采购需求到入库结算的完整采购流程。该模块需要紧密结合面料行业特色，支持坯布、染料、助剂等不同类型物料的采购管理。

### 1.2 业务价值
1. **规范采购流程**：从需求申请到入库结算，全流程标准化管理
2. **降低采购成本**：供应商比价、历史价格分析、批量采购优惠
3. **保证物料质量**：供应商资质管理、来料质检、质量追溯
4. **提高采购效率**：自动补货提醒、采购订单跟踪、异常预警
5. **面料行业适配**：支持坯布采购、染料采购、助剂采购等不同类型

### 1.3 用户角色
| 角色 | 职责 | 使用场景 |
|------|------|----------|
| 采购专员 | 执行采购任务 | 创建采购订单、跟踪订单、处理入库 |
| 采购经理 | 审批采购订单 | 审核采购申请、审批采购合同 |
| 质检员 | 来料质量检验 | 执行质检、记录质检结果 |
| 仓库管理员 | 物料入库管理 | 确认入库数量、办理入库手续 |
| 财务人员 | 采购结算 | 核对发票、安排付款 |

## 二、核心业务流程

### 2.1 采购流程总览
```
采购需求 → 采购申请 → 供应商选择 → 采购订单 → 订单审核 → 
订单跟踪 → 来料质检 → 入库确认 → 发票核对 → 采购结算
```

### 2.2 坯布采购流程（面料行业特色）
```
织造车间需求 → 坯布采购申请 → 供应商报价 → 坯布采购订单 → 
坯布到货 → 坯布质检（检验密度、幅宽、克重） → 坯布入库 → 
按实际检验结果结算（考虑损耗、等级差异）
```

### 2.3 染料助剂采购流程
```
印染车间需求 → 染料助剂申请 → 供应商选择 → 采购订单 → 
到货检验（检验成分、有效期） → 入库（化学品库） → 结算
```

## 三、功能需求详述

### 3.1 采购申请管理

#### 功能描述
各部门根据生产需求提交采购申请，经审批后转为采购订单。

#### 核心字段
**采购申请单主表**：
- 申请单号（自动生成）
- 申请部门
- 申请人
- 申请日期
- 期望到货日期
- 物料类型（坯布/染料/助剂/其他）
- 申请状态（待审核/已审核/已驳回/已转订单）
- 优先级（普通/紧急/特急）
- 备注

**采购申请单明细**：
- 物料 ID
- 物料名称
- 规格型号
- 申请数量
- 单位（米/公斤/升/个）
- 参考单价
- 建议供应商
- 需求说明
- **面料行业特色字段**：
  - 坯布类型（针织/梭织）
  - 纱支规格（如 40S）
  - 密度要求（如 133x72）
  - 幅宽要求（cm）
  - 克重要求（g/m²）
  - 颜色要求（白色/本白/其他）
  - 等级要求（一等品/二等品）

#### 业务规则
1. 采购申请单审核后自动通知采购专员
2. 紧急采购申请优先处理
3. 超过预算金额的申请需要更高级别审批
4. 同一物料可申请合并采购

#### 验收标准
- [ ] 支持在线提交采购申请
- [ ] 支持附件上传（技术图纸、样品照片等）
- [ ] 支持审批流程配置
- [ ] 支持申请单状态跟踪
- [ ] 支持申请单转采购订单

#### 核心字段

- 订单号（自动生成，PO+ 日期 + 流水号）
- 供应商 ID
- 供应商名称
- 订单日期
- 期望到货日期
- 实际到货日期
- 订单状态（待审核/已审核/待到货/部分到货/已完成/已取消）
- 付款方式
- 付款条件
- 交货地址
- 采购员
- 备注
- **面料行业特色字段**：
  - 采购类型（坯布/染料/助剂/其他）
  - 质量等级要求
  - 检验标准
  - 损耗率约定
  - 包装要求

**采购订单明细**：
- 物料 ID
- 物料名称
- 规格型号
- 订单数量
- 已到货数量
- 未到货数量
- 单位
- 单价
- 金额
- 税率
- 含税金额
- **面料行业特色字段**：
  - 坯布纱支
  - 坯布密度
  - 坯布幅宽
  - 坯布克重
  - 颜色要求
  - 等级要求
  - 缸号要求
  - 实际检验结果

#### 业务规则
1. 采购订单必须关联采购申请单（紧急采购除外）
2. 订单金额超过阈值需要经理审批
3. 订单审核后发送给供应商确认
4. 支持订单变更（需审批）
5. 支持分批到货
6. 支持订单关闭和取消

#### 验收标准
- [ ] 支持创建采购订单
- [ ] 支持订单审核流程
- [ ] 支持订单状态跟踪

### 3.3 采购入库管理

#### 功能描述
物料到货后，仓库管理员根据采购订单办理入库手续，质检员执行质量检验。

#### 核心字段
**采购入库单主表**：
- 入库单号（自动生成）
- 采购订单号
- 供应商 ID
- 供应商名称
- 入库日期
- 入库类型（正常入库/退货入库/换货入库）
- 入库状态（待质检/质检合格/质检不合格/已完成）
- 仓库 ID
- 库位 ID
- 收货人
- 质检员
- 备注

**采购入库单明细**：
- 物料 ID
- 物料名称
- 规格型号
- 订单数量
- 实收数量
- 拒收数量
- 单位
- 单价
- 金额
- 批次号（自动生成或手工录入）
- **面料行业特色字段**：
  - 批次号（坯布批次）
  - 色号（如适用）
  - 缸号（如适用）
  - 等级（一等品/二等品/等外品）
  - 检验结果（合格/不合格）
  - 检验说明
  - 实际幅宽
  - 实际克重
  - 实际密度
  - 损耗数量
  - 损耗原因

#### 业务规则
1. 入库单必须关联采购订单
2. 来料必须经过质检才能入库
3. 质检不合格可拒收或特采
4. 面料行业特殊要求：
   - 坯布需检验密度、幅宽、克重
   - 染料需检验成分、有效期
   - 记录批次号，支持质量追溯
5. 入库后自动更新库存

- [ ] 支持扫码入库

## 四、面料行业特色需求

### 4.1 坯布采购特殊要求

#### 质量检验标准
1. **外观检验**：
   - 布面平整度
   - 疵点数量（每百米不超过 X 个）
   - 色差（级数要求）
   - 污渍、破损

2. **物理指标检验**：
   - 纱支偏差（±5% 以内）
   - 密度偏差（±3% 以内）
   - 幅宽偏差（±1cm 以内）
   - 克重偏差（±5g/m²以内）
   - 缩水率（≤3%）

3. **等级判定**：
   - 一等品：所有指标合格
   - 二等品：轻微瑕疵，不影响使用
   - 等外品：严重瑕疵，降级使用或退货

#### 结算规则
1. 按实际检验数量结算
2. 考虑损耗率（一般 3-5%）
3. 等级差异价格调整
4. 质量扣款（如不合格）

### 4.2 染料助剂采购特殊要求

#### 质量检验标准
1. **成分检验**：
   - 有效成分含量
   - 杂质含量
   - pH 值
   - 稳定性

2. **有效期管理**：
   - 生产日期
   - 保质期
   - 存储条件

#### 存储要求
1. 化学品库存储
2. 温湿度控制
3. 危险品管理
4. 先进先出

### 4.3 批次追溯要求

#### 正向追溯
```
供应商 → 采购订单 → 入库单 → 批次号 → 
库存 → 生产领用 → 生产订单 → 成品布批次 → 
销售订单 → 客户
```

#### 反向追溯
```
客户投诉 → 销售订单 → 成品布批次 → 
生产订单 → 领料批次 → 库存批次 → 
采购入库单 → 采购订单 → 供应商
```

## 五、接口需求

### 5.1 与库存管理接口
1. 入库单审核后自动增加库存
2. 支持批次号、色号、缸号传递
3. 支持双计量单位（米/公斤）
4. 支持库存预警触发采购申请

### 5.2 与生产管理接口
1. 接收生产部门的采购申请
2. 采购入库后通知生产部门
3. 支持生产领料追溯

### 5.3 与财务管理接口
1. 采购发票传递到财务
2. 支持采购对账
3. 支持付款申请
4. 成本核算数据提供

### 5.4 与质量管理接口
1. 来料检验标准获取
2. 检验结果记录
3. 不合格品处理
