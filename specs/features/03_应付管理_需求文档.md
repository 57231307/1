# 应付管理模块 - 需求文档

## 一、模块概述

### 1.1 模块定位
应付管理模块是面料 ERP 系统财务管理的核心子模块，与采购管理模块、销售管理模块紧密集成，实现企业对供应商的应付账款全生命周期管理。

### 1.2 业务价值
- **财务透明**：实时掌握应付账款余额和账龄分析
- **资金计划**：为资金安排提供准确数据支持
- **风险控制**：避免逾期付款，维护供应商关系
- **自动化**：减少手工录入，提高财务工作效率

### 1.3 核心功能
1. 应付单管理 - 采购入库自动生成应付
2. 付款申请 - 业务部门发起付款流程
3. 付款执行 - 财务部门执行付款操作
4. 应付核销 - 付款与应付单的核销匹配
5. 账龄分析 - 多维度应付账款分析
6. 供应商对账 - 定期与供应商对账

## 二、业务流程

### 2.1 应付产生流程
```
采购入库单确认
    ↓
自动生成应付单（财务系统）
    ↓
应付会计审核
    ↓
应付单生效（计入应付账款）
```

### 2.2 付款申请流程
```
采购专员发起付款申请
    ↓
采购经理审批
    ↓
财务经理审批
    ↓
总经理审批（超过限额）
    ↓
付款申请生效
```

### 2.3 付款执行流程
```
出纳接收付款申请
    ↓
安排付款（银行转账/承兑汇票）
    ↓
登记付款信息
    ↓
财务经理确认
    ↓
付款完成
```

### 2.4 应付核销流程
```
选择付款单和应付单
    ↓
系统自动匹配核销
    ↓
生成核销记录
    ↓
更新应付余额
```

## 三、功能需求

### 3.1 应付单管理

#### 3.1.1 应付单自动生成
**功能描述**：采购入库单确认后，系统自动生成应付单

**业务规则**：
- 触发时机：采购入库单确认时
- 数据来源：采购入库单、采购订单
- 应付金额 = 入库金额 - 退货金额
- 应付日期 = 入库日期 + 账期天数
- 支持分批入库、汇总生成应付

**数据字段**：
- 应付单号（自动生成：AP20260315001）
- 供应商
- 源单类型（采购入库单）
- 源单号
- 应付日期
- 账期天数
- 到期日期
- 应付金额
- 已付金额
- 未付金额
- 币种
- 业务员
- 制单人
- 审核人
- 状态（草稿、已审核、已核销、部分核销）

#### 3.1.2 应付单手工录入
**功能描述**：支持手工录入非采购产生的应付单

**适用场景**：
- 费用类应付
- 其他应付款
- 调整应付

**数据字段**：
- 应付单号
- 供应商
- 应付类型（采购应付、费用应付、其他应付）
- 费用科目
- 应付日期
- 到期日期
- 应付金额
- 税额
- 备注

#### 3.1.3 应付单审核
**功能描述**：应付会计审核应付单的正确性

**审核要点**：
- 金额是否正确
- 供应商信息是否准确
- 源单是否完整
- 账期是否符合合同

**状态变更**：
- 草稿 → 已审核
- 已审核 → 已核销（全额核销）
- 已审核 → 部分核销（部分核销）

#### 3.1.4 应付单查询
**查询条件**：
- 供应商
- 应付日期范围
- 到期日期范围
- 应付状态
- 业务员
- 金额范围

**列表字段**：
- 应付单号
- 供应商
- 应付日期
- 到期日期
- 应付金额
- 已付金额
- 未付金额
- 状态
- 业务员

### 3.2 付款申请管理

#### 3.2.1 付款申请创建
**功能描述**：采购专员根据应付单创建付款申请

**业务规则**：
- 必须选择已审核的应付单
- 支持多张应付单合并申请
- 申请金额 ≤ 未付金额
- 支持部分付款、全额付款

**数据字段**：
- 申请单号（自动生成：PA20260315001）
- 申请日期
- 供应商
- 付款类型（预付款、进度款、尾款、质保金）
- 付款方式（电汇、承兑汇票、支票、现金）
- 申请金额
- 币种
- 收款账户信息
- 关联应付单列表
- 备注
- 申请人
- 审批状态

#### 3.2.2 付款申请审批
**审批流程**：
1. 采购经理审批 - 业务真实性
2. 财务经理审批 - 资金安排
3. 总经理审批 - 超过限额时

**审批权限**：
- 10 万元以下：财务经理审批
- 10-50 万元：总经理审批
- 50 万元以上：董事长审批

**状态变更**：
- 草稿 → 审批中 → 已批准/已拒绝

#### 3.2.3 付款申请查询
**查询条件**：
- 供应商
- 申请日期范围
- 审批状态
- 申请人
- 金额范围

### 3.3 付款执行管理

#### 3.3.1 付款登记
**功能描述**：出纳根据已批准的付款申请执行付款并登记

**业务规则**：
- 必须选择已批准的付款申请
- 登记实际付款信息
- 支持分批付款
- 支持多种付款方式

**数据字段**：
- 付款单号（自动生成：PAY20260315001）
- 付款日期
- 供应商
- 付款方式
- 付款金额
- 币种
- 汇率
- 手续费
- 实际付款金额
- 银行账号
- 票据号
- 备注
- 出纳
- 付款状态

#### 3.3.2 付款确认
**功能描述**：财务经理确认付款完成

**确认要点**：
- 付款金额是否正确
- 付款信息是否完整
- 银行回单是否上传

**状态变更**：
- 已登记 → 已确认

#### 3.3.3 付款查询
**查询条件**：
- 供应商
- 付款日期范围
- 付款方式
- 付款状态
- 出纳

### 3.4 应付核销管理

#### 3.4.1 自动核销
**功能描述**：系统自动匹配付款单和应付单进行核销

**核销规则**：
- 同供应商优先
- 到期日优先（先到期先核销）
- 金额匹配
- 支持部分核销

**核销结果**：
- 生成核销记录
- 更新应付单已付金额
- 更新付款单已核销金额

#### 3.4.2 手工核销
**功能描述**：支持手工指定核销关系

**适用场景**：
- 特殊业务需要
- 自动核销错误时调整

**操作流程**：
1. 选择付款单
2. 选择应付单（可多选）
3. 输入核销金额
4. 确认核销

#### 3.4.3 核销查询
**查询条件**：
- 供应商
- 核销日期范围
- 核销类型（自动/手工）

### 3.5 账龄分析

#### 3.5.1 账龄分析表
**分析维度**：
- 供应商
- 业务员
- 部门
- 账龄区间

**账龄区间**：
- 未到期
- 逾期 1-30 天
- 逾期 31-60 天
- 逾期 61-90 天
- 逾期 91-180 天
- 逾期 180 天以上

**统计指标**：
- 应付总额
- 已付金额
- 未付金额
- 逾期金额
- 各账龄区间金额

#### 3.5.2 应付余额表
**统计维度**：
- 供应商
- 月份
- 业务员

**统计指标**：
- 期初余额
- 本期增加
- 本期减少
- 期末余额

### 3.6 供应商对账

#### 3.6.1 对账单生成
**功能描述**：定期生成供应商对账单

**对账周期**：
- 按月对账
- 按季度对账
- 按需对账

**对账内容**：
- 期初应付
- 本期入库
- 本期退货
- 本期付款
- 期末应付

#### 3.6.2 对账单确认
**功能描述**：与供应商确认对账单

**确认方式**：
- 邮件确认
- 传真确认
- 纸质盖章确认

**确认状态**：
- 待确认
- 已确认
- 有差异

## 五、接口需求

- 应付单 → 生成会计凭证

### 5.3 与资金管理接口
- 付款申请 → 资金计划
- 付款执行 → 资金支付

## 八、业务场景示例

### 场景 1：采购应付
1. 仓库确认采购入库单（10000 元）
2. 系统自动生成应付单（账期 30 天）
3. 30 天后到期，采购专员发起付款申请
4. 财务经理审批通过
5. 出纳登记付款并确认
6. 系统自动核销应付单
7. 应付单状态更新为"已核销"

### 场景 2：预付款
1. 签订采购合同，需预付 30%
2. 采购专员创建预付款申请（关联采购订单）
3. 总经理审批通过（金额 5 万元）
4. 出纳登记预付款
5. 货物入库后，预付款自动冲抵应付

### 场景 3：账龄分析
1. 财务人员选择账龄分析表
2. 设置分析日期（2026-03-15）
3. 系统显示各供应商应付账款账龄分布
4. 发现 A 供应商逾期 180 天以上 10 万元
5. 通知采购专员跟进处理
