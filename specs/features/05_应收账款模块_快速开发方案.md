## 一、核心业务对比

| 业务场景 | 应付账款（已完成） | 应收账款（待开发） |
|---------|------------------|------------------|
| **业务方向** | 采购付款 | 销售回款 |
| **往来对象** | 供应商 | 客户 |
| **单据来源** | 采购入库单、采购退货单 | 销售出库单、销售退货单 |
| **发票类型** | 采购发票（进项） | 销售发票（销项） |
| **收款方式** | 银行转账、承兑汇票 | 银行转账、承兑汇票 |
| **核销逻辑** | 付款核销应付单 | 收款核销应收单 |
| **对账** | 与供应商对账 | 与客户对账 |

## 二、数据库设计（参考应付模块）

### 核心表结构

1. **应收单表** (`ar_invoices`) - 对应 `ap_invoices`
   - 来源：销售出库单、销售退货单
   - 关联：客户 ID、销售订单号
   - 字段：应收金额、已收金额、未收金额

2. **收款申请单表** (`ar_collection_requests`) - 对应 `ap_payment_requests`
   - 申请收款（业务员发起）
   - 多级审批（按金额）

3. **收款单表** (`ar_collections`) - 对应 `ap_payments`
   - 实际收款记录
   - 收款方式、收款账户

4. **核销记录表** (`ar_verifications`) - 对应 `ap_verifications`
   - 收款核销应收单
   - 自动核销（到期日优先）

5. **对账单表** (`ar_reconciliations`) - 对应 `ap_reconciliations`
   - 与客户对账
   - 确认/争议处理

6. **报表表** (`ar_reports`) - 对应 `ap_reports`
   - 账龄分析
   - 收款统计

## 四、快速开发策略

### 策略 1：复制应付模块代码，然后适配

**步骤**:
1. 复制 `ap_*.rs` 文件，重命名为 `ar_*.rs`
2. 全局替换：
   - `ap_` → `ar_`
   - `应付` → `应收`
   - `付款` → `收款`
   - `供应商` → `客户`
   - `采购` → `销售`
   - `payment` → `collection`
   - `supplier` → `customer`

3. 调整业务逻辑：
   - 应收单从销售出库单生成（不是采购入库单）
   - 收款申请由业务员发起（不是采购员）
   - 核销逻辑相同（保持算法）

### 策略 2：直接生成完整代码

**优势**: 代码更贴合应收账款场景  
**劣势**: 开发时间稍长（约 2 小时）
