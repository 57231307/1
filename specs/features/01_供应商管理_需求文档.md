## 一、模块概述

### 1.1 模块定位
供应商管理模块是采购管理的基础，用于管理面料企业的供应商信息、供应商评估、供应商分级等核心功能。该模块与采购管理、应付管理、库存管理紧密集成，形成完整的供应链闭环。

### 1.2 业务价值
- **规范供应商管理**：建立统一的供应商信息库
- **降低采购风险**：通过供应商评估和分级，优选供应商
- **提高采购效率**：快速定位合格供应商
- **质量追溯**：质量问题可追溯到具体供应商

### 1.3 核心功能
1. 供应商基本信息管理
2. 供应商分类管理
3. 供应商分级管理
4. 供应商联系人管理
5. 供应商评估体系
6. 供应商资质管理
7. 供应商黑名单管理

## 二、用户角色与使用场景

### 2.1 用户角色

#### 采购专员
- **职责**：维护供应商信息、发起供应商评估
- **核心需求**：
  - 快速录入新供应商
  - 查询供应商历史交易记录
  - 发起供应商评估流程

#### 采购经理
- **职责**：审核供应商资质、审批供应商分级
- **核心需求**：
  - 查看供应商评估报告
  - 审批供应商等级调整
  - 供应商绩效考核

#### 质量管理员
- **职责**：记录供应商质量问题、参与供应商评估
- **核心需求**：
  - 记录供应商来料不良
  - 查看供应商质量统计
  - 发起供应商质量改进

#### 仓库管理员
- **职责**：记录供应商来料情况
- **核心需求**：
  - 查询供应商供货及时率
  - 记录供应商包装规范度

### 3.3 供应商分级管理

#### 3.3.1 供应商等级
**功能描述**：根据供应商综合表现进行分级管理

**等级体系**：
| 等级 | 名称 | 评分范围 | 说明 | 权限 |
|------|------|----------|------|------|
| A | 战略供应商 | 90-100 分 | 优秀供应商，优先合作 | 优先采购、免检、月结 |
| B | 合格供应商 | 75-89 分 | 良好供应商，正常合作 | 正常采购、抽检、月结 |
| C | 考察供应商 | 60-74 分 | 合格供应商，限制合作 | 限制采购、全检、现结 |
| D | 不合格供应商 | 60 分以下 | 不合格，淘汰 | 暂停采购、列入黑名单 |

**字段清单**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 等级编码 | String(10) | ✓ | A/B/C/D |
| 等级名称 | String(50) | ✓ | 战略供应商/合格供应商等 |
| 最低分数 | Decimal(5,2) | ✓ | 该等级的最低分 |
| 最高分数 | Decimal(5,2) | ✓ | 该等级的最高分 |
| 颜色标识 | String(20) | ✓ | 绿/蓝/黄/红 |
| 权限说明 | Text | ✓ | 该等级的权限描述 |
| 是否启用 | Boolean | ✓ | 默认 true |

**业务规则**：
1. 等级评定基于供应商评估分数
2. 等级调整需要采购经理审批
3. 降级时自动通知供应商
4. 连续 3 次评估为 D 级，自动列入黑名单

### 3.4 供应商评估体系

#### 3.4.1 评估指标
**功能描述**：建立供应商综合评估指标体系

**评估维度**（总分 100 分）：

**1. 质量水平（35 分）**
- 来料合格率（15 分）
  - 计算公式：合格批次/总批次 × 100%
  - 评分：≥98% 得 15 分，≥95% 得 12 分，≥90% 得 8 分，<90% 得 0 分
- 质量事故次数（10 分）
  - 评分：0 次得 10 分，1 次得 5 分，≥2 次得 0 分
- 质量改进响应（10 分）
  - 评分：及时改进得 10 分，一般得 5 分，不配合得 0 分

**2. 交货能力（25 分）**
- 交货及时率（15 分）
  - 计算公式：及时批次/总批次 × 100%
  - 评分：≥95% 得 15 分，≥90% 得 12 分，≥85% 得 8 分，<85% 得 0 分
- 订单完成率（10 分）
  - 计算公式：完成订单数/总订单数 × 100%
  - 评分：≥98% 得 10 分，≥95% 得 8 分，≥90% 得 5 分，<90% 得 0 分

**3. 价格水平（20 分）**
- 价格竞争力（10 分）
  - 评分：低于市场价得 10 分，持平得 6 分，高于市场得 0 分
- 价格稳定性（10 分）
  - 评分：半年未涨价得 10 分，涨价 1 次得 5 分，涨价≥2 次得 0 分

**4. 服务水平（15 分）**
- 响应速度（5 分）
  - 评分：2 小时内响应得 5 分，24 小时内得 3 分，>24 小时得 0 分
- 售后服务（5 分）
  - 评分：优秀得 5 分，良好得 3 分，差得 0 分
- 配合度（5 分）
  - 评分：积极配合得 5 分，一般得 3 分，不配合得 0 分

**5. 技术能力（5 分）**
- 研发能力（3 分）
  - 评分：强得 3 分，一般得 2 分，弱得 0 分
- 技术支持（2 分）
  - 评分：优秀得 2 分，良好得 1 分，差得 0 分

**业务规则**：
1. 评估周期：季度评估（每季度一次）
2. 评估发起：采购专员发起，多部门参与评分
3. 评估流程：
   ```
   发起评估 → 各部门评分 → 汇总分数 → 确定等级 → 审批 → 通知供应商
   ```
4. 评估结果自动计算供应商等级
5. 评估历史永久保存

---

#### 3.4.2 评估流程
**功能描述**：供应商评估的完整流程管理

**参与部门**：
- 采购部（主导）
- 质量部（质量评分）
- 仓库（交货评分）
- 生产部（使用反馈）
- 财务部（价格评分）

**评估步骤**：
1. **评估发起**（采购专员）
   - 选择评估供应商
   - 选择评估季度
   - 设置评估参与人
   - 提交评估申请

2. **各部门评分**（评估参与人）
   - 质量部：填写质量数据，系统自动计算得分
   - 仓库：填写交货数据，系统自动计算得分
   - 生产部：填写使用反馈
   - 财务部：填写价格数据

3. **分数汇总**（系统自动）
   - 汇总各维度得分
   - 计算总分
   - 确定等级
   - 生成评估报告

4. **评估审批**（采购经理）
   - 查看评估报告
   - 审批等级调整
   - 填写审批意见

5. **结果通知**（采购专员）
   - 发送评估结果给供应商
   - 记录供应商反馈
   - 归档评估资料

### 3.5 供应商资质管理

#### 3.5.1 资质类型
**功能描述**：管理供应商提供的各类资质证书

**资质类型**：
1. **营业执照**（必须）
   - 扫描件
   - 有效期
   - 年检记录

2. **税务登记证**（必须）
   - 扫描件
   - 纳税人类型

3. **组织机构代码证**（必须）
   - 扫描件
   - 有效期

4. **质量体系认证**（可选）
   - ISO9001 认证
   - ISO14000 认证
   - Oeko-Tex Standard 100 认证

5. **产品检测报告**（可选）
   - 第三方检测报告
   - 有效期

**字段清单**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 资质名称 | String(200) | ✓ | 营业执照等 |
| 资质类型 | Enum | ✓ | 营业执照/税务登记证等 |
| 资质编号 | String(100) | ✓ | 证书编号 |
| 发证机构 | String(200) | ✓ | 发证单位 |
| 发证日期 | Date | ✓ | 证书签发日期 |
| 有效期至 | Date | ✓ | 证书有效期 |
| 附件 | String(500) | ✓ | 扫描件上传路径 |
| 是否年检 | Boolean | ✗ | 是否需要年检 |
| 年检记录 | Text | ✗ | 年检历史记录 |
| 备注 | Text | ✗ | 其他说明 |

**业务规则**：
1. 营业执照、税务登记证、组织机构代码证为必须资质
2. 资质到期前 30 天自动提醒
3. 资质过期自动停用供应商
4. 支持多个资质证书

### 3.6 供应商黑名单管理

#### 3.6.1 黑名单条件
**功能描述**：将不合格供应商列入黑名单

**列入条件**（满足任一条件）：
1. 连续 3 次评估为 D 级
2. 发生重大质量事故（损失≥10 万元）
3. 提供虚假资质
4. 恶意欺诈行为
5. 严重违反合同
6. 商业贿赂行为
7. 其他严重违规行为

**字段清单**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 供应商 ID | Integer | ✓ | 关联供应商 |
| 列入日期 | Date | ✓ | 列入黑名单日期 |
| 列入原因 | Enum | ✓ | 质量事故/欺诈/违约等 |
| 详细说明 | Text | ✓ | 具体事实描述 |
| 证据材料 | Text | ✓ | 相关证据 |
| 审批人 | Integer | ✓ | 审批领导 ID |
| 审批日期 | Date | ✓ | 审批日期 |
| 是否永久 | Boolean | ✓ | 永久/有期限 |
| 解禁日期 | Date | ✗ | 允许解禁的日期 |
| 解禁条件 | Text | ✗ | 需要满足的条件 |
| 备注 | Text | ✗ | 其他说明 |

**业务规则**：
1. 列入黑名单需要采购总监审批
2. 黑名单供应商自动停用
3. 黑名单供应商禁止创建采购订单
4. 永久黑名单不可解禁
5. 有期限黑名单到期可申请解禁

---

#### 3.6.2 黑名单解禁
**功能描述**：符合条件的黑名单供应商可申请解禁

**解禁条件**：
1. 达到解禁期限（非永久黑名单）
2. 完成整改
3. 提供改进证据
4. 通过重新评估

**解禁流程**：
```
供应商申请 → 采购专员审核 → 重新评估 → 采购经理审批 → 移出黑名单
```

## 五、业务流程

### 5.1 供应商新增流程
```
采购专员发起 → 录入供应商信息 → 上传资质文件 → 
采购经理审核 → 审核通过 → 分配供应商编码 → 
录入联系人 → 完成
```

### 5.2 供应商评估流程
```
采购专员发起（季度） → 选择评估供应商 → 
各部门评分（质量/仓库/生产/财务） → 
系统汇总分数 → 自动计算等级 → 
采购经理审批 → 通知供应商 → 归档
```

### 5.3 供应商降级流程
```
评估分数下降 → 系统自动预警 → 
采购专员发起降级申请 → 采购经理审批 → 
通知供应商 → 更新供应商等级 → 调整采购策略
```

### 5.4 供应商黑名单流程
```
发现违规行为 → 采购专员收集证据 → 
采购经理审核 → 采购总监审批 → 
列入黑名单 → 停用供应商 → 通知相关部门
```

## 六、报表与统计

### 6.1 供应商汇总表
**报表内容**：
- 供应商总数（按状态统计）
- 供应商总数（按类型统计）
- 供应商总数（按等级统计）
- 新增供应商数量（按月）
- 停用供应商数量（按月）

- 平均分最低的供应商 Top10

- 供应商综合得分排名

**报表内容**：
- 资质即将到期清单（30 天内）

- 未年检资质清单

## 七、与其他模块的集成关系

### 7.1 与采购管理的集成
- 采购订单必须选择已审核的供应商
- 采购订单自动记录供应商交货及时率
- 采购退货自动记录供应商质量问题

### 7.2 与库存管理的集成
- 采购入库自动关联供应商
- 来料检验记录供应商质量数据
- 库存调整可追溯到供应商

### 7.3 与财务管理的集成
- 供应商应付账款管理
- 供应商付款记录
- 供应商发票管理

### 7.4 与质量管理的集成
- 来料检验记录供应商质量
- 质量事故记录供应商责任
- 质量改进跟踪供应商响应
