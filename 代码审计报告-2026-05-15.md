## 🔴 严重级别问题（必须立即修复）

### 1. 文件上传路径遍历漏洞

**位置**: `backend/src/handlers/system_update_handler.rs:152-184`  
**严重程度**: 🔴 严重  
**CVSS 估算**: 8.1 (High)

**问题描述**:
系统更新功能在上传和解压 ZIP 文件时存在严重的路径遍历漏洞：
1. 直接使用用户提供的文件名，未进行安全过滤
2. ZIP 解压时未验证文件路径是否在目标目录内
3. 未验证上传文件的实际内容类型

**攻击场景**:
攻击者可构造包含 `../../../etc/passwd` 路径的恶意 ZIP 文件，覆盖系统关键文件，导致 RCE（远程代码执行）。

**修复建议**:
```rust
// 1. 生成随机文件名替代用户提供的文件名
let safe_filename = format!("update_{}.zip", uuid::Uuid::new_v4());
let save_path = temp_dir.join(safe_filename);

// 2. 验证文件类型（Magic Bytes）
fn verify_zip_magic(data: &[u8]) -> bool {
    data.starts_with(&[0x50, 0x4B, 0x03, 0x04])  // PK\x03\x04
}

// 3. 解压时验证路径
fn is_safe_path(base: &Path, path: &Path) -> bool {
    path.starts_with(base)
}

// 4. 限制文件大小
const MAX_UPDATE_SIZE: u64 = 100 * 1024 * 1024; // 100MB
```

**优先级**: P0 - 立即修复（24 小时内）

### 2. JWT 密钥强度检查不足

**当前代码**:

**修复建议**:
```rust
// 添加运行时密钥强度验证
fn validate_jwt_secret_at_runtime(secret: &str) -> bool {
    secret.len() >= 32 && 
    !secret.contains("change-in-production") &&
    !secret.contains("change-this") &&
    !secret.contains("local-dev") &&
    calculate_entropy(secret) > MIN_ENTROPY_THRESHOLD
}

// 生产环境强制独立 cookie_secret
if env == "production" && settings.auth.cookie_secret.is_none() {
```

### 3. Token 存储方式不安全（前端）

**位置**: `frontend/src/utils/storage.ts`, `frontend/src/router/index.ts`  
**严重程度**: 🔴 严重  
**CVSS 估算**: 7.2 (High)

**问题描述**:
使用 `localStorage` 存储认证 Token 存在严重 XSS 风险：
- 任何 XSS 漏洞都可能导致 Token 被窃取
- Token 不会自动过期
- 同域下所有页面均可访问

**修复建议**:
```typescript
// 方案 1: 使用 sessionStorage（推荐）
const TOKEN_KEY = 'access_token'

export function getToken(): string | null {
  return sessionStorage.getItem(TOKEN_KEY)
}

export function setToken(token: string): void {
  sessionStorage.setItem(TOKEN_KEY, token)
}

// 方案 2: 使用 HttpOnly Cookie（最佳）
// 后端设置 Cookie，前端无法通过 JS 访问
```

**优先级**: P0 - 立即修复（48 小时内）

### 5. Token 刷新机制缺失（前端）

**位置**: `frontend/src/api/auth.ts`, `frontend/src/api/request.ts`  
**严重程度**: 🟠 高  
**CVSS 估算**: 5.3 (Medium)

**问题描述**:
虽然定义了 `refreshToken` 函数，但在 `request.ts` 的拦截器中**未实现自动刷新逻辑**。当 Token 过期时，只会跳转到登录页，用户体验差且可能导致数据丢失。

**修复建议**:
```typescript
// 实现无感刷新逻辑
let isRefreshing = false
let refreshSubscribers: ((token: string) => void)[] = []

function subscribeTokenRefresh(cb: (token: string) => void) {
  refreshSubscribers.push(cb)
}

function onTokenRefreshed(token: string) {
  refreshSubscribers.forEach(cb => cb(token))
  refreshSubscribers = []
}

// 在响应拦截器中
if (error.response?.status === 401 && !originalRequest._retry) {
  if (isRefreshing) {
    return new Promise((resolve) => {
      subscribeTokenRefresh((token) => {
        originalRequest.headers.Authorization = `Bearer ${token}`
        resolve(this.instance(originalRequest))
      })
    })
  }
  
  originalRequest._retry = true
  isRefreshing = true
  
  try {
    const refreshToken = getRefreshToken()
    const { data } = await refreshToken(refreshToken)
    setToken(data.token)
    onTokenRefreshed(data.token)
    return this.instance(originalRequest)
  } finally {
    isRefreshing = false
  }
}
```

**优先级**: P1 - 近期修复（1 周内）

### 6. Webhook URL 未验证 SSRF 风险

**位置**: `backend/src/services/webhook_service.rs`  
**严重程度**: 🟠 高

**问题描述**:
- 未验证 URL 是否指向内网地址（SSRF 风险）
- 未验证 URL 协议（可能是 `file://`, `gopher://` 等）

**修复建议**:
```rust
fn validate_webhook_url(url: &str) -> Result<(), ValidationError> {
    let parsed = Url::parse(url)?;
    
    // 仅允许 HTTP/HTTPS
    if !matches!(parsed.scheme(), "http" | "https") {
        return Err(ValidationError::new("invalid_scheme"));
    }
    
    // 检查是否内网 IP
    let ip = lookup_host(parsed.host_str().ok_or("no_host")?)?;
    if is_private_ip(ip) {
        return Err(ValidationError::new("private_ip_not_allowed"));
    }
    
    Ok(())
}
```

```rust
    query.map(|q| {
        q.split('&')
            .map(|pair| {
                if pair.starts_with("password=") || pair.starts_with("token=") {
                    pair.split('=').next().map(|k| format!("{}=***", k)).unwrap_or_default()

                }
            })
            .collect::<Vec<_>>()
            .join("&")
    }).unwrap_or_default()
}
```

### 9. 缺乏输入验证（前端）

**位置**: `frontend/src/views/system/index.vue`, `frontend/src/views/Login.vue`  
**严重程度**: 🟠 高

**问题描述**:
登录表单仅有基本必填验证，系统管理页面的用户/角色/部门表单缺少：
- 邮箱格式验证
- 手机号格式验证
- 密码强度验证
- 特殊字符过滤

**修复建议**:
```typescript
const validateEmail = (rule: any, value: string, callback: any) => {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
  if (value && !emailRegex.test(value)) {
    callback(new Error('请输入有效的邮箱地址'))
  } else {
    callback()
  }
}

const validatePassword = (rule: any, value: string, callback: any) => {
  if (value && value.length < 8) {
    callback(new Error('密码长度至少 8 位'))
  } else if (value && !/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).+$/.test(value)) {
    callback(new Error('密码需包含大小写字母和数字'))
  } else {
    callback()
  }
}
```
