## 三、更新模式

### 标准更新步骤

每个 Handler 函数按照以下模式更新：

#### 1. 添加导入

```rust
use crate::middleware::auth_context::AuthContext;
use tracing::{info, warn, error};
```

#### 2. 添加 AuthContext 参数

```rust
pub async fn some_handler(
    // ... 其他参数
    auth: AuthContext,  // 必须是最后一个参数
) -> Result<Json<ApiResponse<T>>, AppError> {
    // ...
}
```

#### 3. 添加操作日志

```rust
// 开始操作
info!("用户 {} 执行操作，参数：{}", auth.username, some_param);

// 验证错误
req.validate().map_err(|e| {
    warn!("用户 {} 验证失败：{}", auth.username, e);
    AppError::ValidationError(e.to_string())
})?;

// 业务操作（使用 auth.user_id）
let result = service.some_operation(param, auth.user_id).await?;

// 操作成功
info!("用户 {} 操作成功：{}", auth.username, result.some_field);
```

### 日志级别说明

- **info**: 正常操作记录（查询、创建、更新、删除、审核等）
- **warn**: 可恢复的错误（验证失败、权限不足等）
- **error**: 严重错误（数据库错误、系统错误等）

### 日志内容规范

**查询操作**:
```rust
info!("用户 {} 查询{}列表，参数：{:?}", auth.username, 模块名，params);
info!("查询成功，共 {} 条记录", total);
```

**创建操作**:
```rust
info!("用户 {} 创建{}，关键参数：{}", auth.username, 模块名，key_param);
info!("用户 {} 创建{}成功：{}", auth.username, 模块名，result.no);
```

**更新操作**:
```rust
info!("用户 {} 更新{} ID: {}", auth.username, 模块名，id);
info!("用户 {} 更新{}成功：{}", auth.username, 模块名，result.no);
```

**删除操作**:
```rust
info!("用户 {} 删除{} ID: {}", auth.username, 模块名，id);
info!("用户 {} 删除{}成功", auth.username, 模块名);
```

**审核/审批操作**:
```rust
info!("用户 {} 审核{} ID: {}", auth.username, 模块名，id);
info!("用户 {} 审核{}成功：{}", auth.username, 模块名，result.no);
```

## 四、认证中间件配置

```rust
// 在 routes/mod.rs 中

    .nest("/invoices", ap_invoice_routes)
    .nest("/payment-requests", ap_payment_request_routes)
    .nest("/payments", ap_payment_routes)
    .nest("/verifications", ap_verification_routes)
    .nest("/reconciliations", ap_reconciliation_routes)
```

## 六、日志配置

确保在 `main.rs` 中配置了 tracing：

```rust
use tracing_subscriber::{fmt, prelude::*, EnvFilter};

// 初始化日志
tracing_subscriber::registry()
    .with(fmt::layer())
    .with(EnvFilter::from_default_env())
    .init();
```

环境变量设置日志级别：
```bash
RUST_LOG=info cargo run
```
